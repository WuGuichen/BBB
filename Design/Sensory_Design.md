# 1. 三层信息模型

| 层 | 内容 | 玩家可见？ |
|---|------|-----------|
| **真相层** | 地图实际有什么 | ✗ 编辑器视图 |
| **生物认知层** | 生物通过传感器形成的 Belief | 部分可见（通过传感器揭示） |
| **玩家认知层** | 玩家看到什么 | 可见（通过渲染层） |

**三层的"差异"就是策略的核心战场**：

| 场景 | 真相 | 生物 Belief | 玩家看到 | 体验 |
|------|------|------------|---------|------|
| 双盲 | 敌人在前方 | 无 | 无 | 紧张 |
| 嗅觉检测 | 敌人在前方 | Threat, Confidence 0.6, 位置模糊 | 气味流方向 | 犹豫（有东西，不知道是什么） |
| 视觉发现 | 敌人在前方 | Threat, Confidence 1.0, 精确位置 | 实景 | 决断（打还是跑？） |

## 1.1 记忆衰减

`MVP`

对应 [System_Core §5](System_Core.md#5-信念生命周期) 的"过期残影"。

| 状态 | 视觉 | 对应 System_Core |
|------|------|-----------------|
| 当前可见 | 全亮，颜色饱和 | Belief.Confidence 高 |
| 曾经可见但不在视野 | 颜色冷掉 60%，动态实体不更新位置 | Belief 衰减中，残影显示最后已知位置 |
| 嗅觉感知但未视觉揭开 | 颜色暗 + 气味流动粒子 | Belief 来自嗅觉通道，Confidence 较低 |
| 完全未知 | 黑/雾 | 无 Belief |

**"你以为它还在原地，结果它已经摸过来了"**——这是 Belief 过期的视觉表达。

# 2. 感知模态的视觉语言

## 2.1 视觉 `MVP`

对应 System_Core 的 `StimChannel.Visual`。

**揭示方式**：以生物为中心，圆形范围揭开迷雾。`GivesPosition=true, GivesIdentity=true`。

**视觉语言**：实景揭开。迷雾消失，物体显现。

**局限**：黑暗区域 Range 缩小。被遮挡的物体看不到。

## 2.2 嗅觉 `MVP`

对应 System_Core 的 `StimChannel.Smell`。

**揭示方式**：显示气味流向，但不显示精确位置。`GivesPosition=false, GivesIdentity=false`。

**视觉语言**：
```
远处有威胁 → 红色气流从某个方向飘来（强度由透明度表示）
附近有资源 → 绿色气流从某个方向飘来
没有气味 → 无显示
```

**精度**：方向 ±30° 误差，距离只有远/中/近三档。

**这就是"错误信念"的来源**——嗅觉告诉你"某个方向有威胁"，但可能是清道夫（不是威胁）。传感器不给 identity，生物只能猜。

## 2.3 听觉 `Post-MVP`

对应 System_Core 的 `StimChannel.Sound`。

**视觉语言**：同心圆涟漪。`GivesPosition=false, GivesIdentity=false`。

## 2.4 震动/热/电磁 `Long-term`

后期扩展通道。

# 3. 每种感知产生不同的 Belief

感知不是"揭开地图"。每种 SensorDef 产生不同的 Belief，喂进 System_Core §6 的效用公式：

| SensorDef | Belief 特点 | 效用影响 |
|-----------|------------|---------|
| 视觉 | Confidence 高，位置精确，有 identity | 直接喂进 Relevance |
| 嗅觉 | Confidence 中，位置模糊，无 identity | 间接影响（可能误判） |
| 听觉 | Confidence 低，无位置，无 identity | 弱影响 |

**同一刺激 + 不同 SensorDef = 不同 Belief = 不同行为。** 这就是"换部件换行为"的根。

# 4. AI 反馈四层

玩家"看懂系统在干什么"不是一件事，是四件事。每一层对应 System_Core 的不同阶段。

## 4.1 行为层 `MVP` — 对应 Utility 输出

生物的物理动作 = argmax(Utility) 的结果。**无 UI 也能读。**

| 状态 | 物理表现 |
|------|---------|
| 探索 | 匀速前进，朝向微扫 |
| 警戒 | 减速 50%，朝向锁定 |
| 犹豫 | 停下，小幅左右晃动 |
| 冲刺 | 加速 30%，朝向锁定 |
| 后退 | 倒退，朝向仍对威胁 |
| 拾取 | 停下，弯腰 0.5s |

**占位阶段**：圆柱体 + 朝向箭头 + 速度变化。

**程序化身体表现**（对应 [GDD §3.3.6](GDD.md#336-部件语法程序化建模)）：

| Utility 输出 | 身体表现 | 部件损伤时 |
|-------------|---------|-----------|
| Explore | 匀速前进，头/传感器轻微扫动 | 腿 Weakened → 拖行，速度下降 |
| Alert | 减速，传感器朝向 Belief.EstimatedPos | 眼 Misreading → 扫描更频繁 |
| Hesitate | 停下，左右小幅晃动，两个候选方向拉扯身体 | — |
| Attack | 核心前倾，武器部件朝目标伸出 | 爪 Disabled → 还会冲，但打不过 |
| Flee | 身体反向移动，头/传感器仍对威胁 | 腿 Weakened → 跑不掉 |
| Forage | 嘴/爪朝资源方向伸出，路径弯曲 | — |

**犹豫的程序化表现**：AttackScore 72 vs FleeScore 68 → 核心朝敌人倾斜 + 后腿往后拖 + 头顶 `?` + 爪子抬起但没冲出去。**玩家不看数字也能理解：它想打，又想跑。**

## 4.2 情绪层 `MVP` — 对应 Belief 状态

头顶图标传达 **Belief 的 confidence 和冲突程度**：

| 情绪 | 头顶 | 含义 |
|------|------|------|
| 好奇 | `?` 缓慢飘出 | 有低 confidence Belief |
| 紧张 | `!` 闪烁 | 有高 confidence 威胁 Belief |
| 害怕 | `!!` + 抖动 | 多个威胁 Belief 叠加 |
| 兴奋 | `★` | 有高价值 Belief（资源/猎物） |

**情绪图标必须早于行为变化 0.3-0.5 秒出现**——这是"预兆"。

## 4.3 理由层 `MVP` — 对应 Utility 拆解

默认只显示行为，**按 R 键** 显示 Utility 拆解：

```
按 R：
→ AttackScore = 72 (Relevance×Confidence×NeedWeight×PersonalityBias)
→ FleeScore = 68
→ 差距 4 < 阈值 15 → 犹豫
```

**每一项都可追溯 → 满足铁律 A。**

## 4.4 结果层 `MVP` — 对应动作→刺激映射

信号发出后的反馈：

| 情况 | 表现 |
|------|------|
| 信号被执行 | 行为改变（转向/减速/加速） |
| 信号被压制 | 头顶 `✗`，行为不变（PersonalityResistance 压过信号权重） |
| 信号导致后果 | 后果通过总线传播（可追溯） |

**没有"它点了点头"这种拟人反馈。** 只有行为变化和 Utility 数字。

# 5. 犹豫的表现规范

对应 [System_Core §6](System_Core.md#6-效用与犹豫决策且可归因) 的犹豫触发条件。

## 5.1 视觉表现

```
0.0s  头顶 "?" 淡入（情绪层先到）
0.2s  减速至停止（行为层）
0.3s  Brain View 浮现两个候选 Intent + 各自 Utility 分数（理由层）
0.3s  画面慢放 0.7x，相机微推 1.1x
窗口  3 秒。期间玩家可发信号。
3.0s+ 若无干预，自动选 argmax，恢复速度
```

## 5.2 触发条件

```
最高 Utility 与次高差距 < 15
且 最高分 > 30
且 距上次犹豫 > 5s
```

## 5.3 反向高亮

犹豫结束后，显示 **落选 Intent + 落选原因（哪个因子输了）**：

```
选择了 AttackScore 72
未选择 FleeScore 68 — FleeScore 的 Confidence 较低（嗅觉 Belief 不确定）
```

**"它本可以做什么但没做"是最有力的认知反馈。** 对应铁律 A：可归因。

# 6. 信息密度

**密度由系统保证，不是设计师埋点。**

信息来源是房间里活物的自然行为——它们发射 Stimulus，被传感器读取，形成 Belief，产生行为，再发射 Stimulus。闭环自己跑。

**设计师管"放多少活物"，不管"什么时候发生什么事"。**

详见 [World_Simulation §7](World_Simulation.md#7-活物密度规范)。
