# SENSORY DESIGN — 感知与未知设计规范

> 版本 0.8 | 2026-05-27
>
> 这是项目的核心设计文档。"通过他者的感知理解未知"是游戏的核心动词，感知系统的设计质量决定项目成败。

---

# 1. 三层信息模型

| 层 | 内容 | 玩家可见？ |
|---|------|-----------|
| **真相层** | 地图实际有什么（敌人/资源/危险/宝箱） | ✗ 编辑器视图 |
| **生物认知层** | 生物通过传感器知道什么 | 部分可见（通过传感器揭示） |
| **玩家认知层** | 玩家看到什么 | 可见（通过生物的眼睛 + 已揭开区域 + 侦察道具） |

**三层的"差异"就是张力的来源**：

| 场景 | 真相 | 生物知道 | 玩家知道 | 体验 |
|------|------|----------|----------|------|
| 双盲 | 敌人在前方 | 不知道 | 不知道 | 紧张（"前面有东西吗？"） |
| 侦察探针 | 敌人在前方 | 不知道 | 知道 | 焦虑（"它要踩雷了！"） |
| 嗅觉检测 | 敌人在前方 | 闻到了 | 闻到了 | 犹豫（"有东西，但不知道在哪"） |
| 视觉发现 | 敌人在前方 | 看到了 | 看到了 | 决断（"打还是跑？"） |

# 2. 感知模态

## 2.1 视觉 `MVP`

生物的"眼"。最基础的感知。

**揭示方式**：以生物为中心，圆形范围揭开迷雾。看到 = 知道位置、类型、状态。

**视觉语言**：实景揭开。迷雾消失，物体显现。

**局限**：黑暗区域视觉范围缩小或无效。被遮挡的物体看不到。

## 2.2 嗅觉 `MVP`

生物的"鼻"。能发现看不到的东西。

**揭示方式**：显示**气味流向**（彩色气流/箭头指向气味来源），但**不显示具体位置和形状**。

**视觉语言**：
```
远处有威胁 → 红色气流从某个方向飘来
附近有资源 → 绿色气流从某个方向飘来
没有气味 → 无显示
```

**关键差异**：视觉说"那里有一只敌人"，嗅觉说"某个方向有威胁"。**信息精度不同。**

**行为影响**：嗅觉检测到威胁时，AI 产生 `threat.smell` fact（位置不精确，confidence 中等）。AI 可能选择绕路，也可能选择调查。

## 2.3 听觉 `Post-MVP`

生物的"耳"。能听到声音但不知道在哪。

**揭示方式**：显示**声波涟漪**（从某个大致方向传来），有音量大小但**无精确方位**。

**视觉语言**：
```
远处有脚步声 → 小涟漪
近处有碰撞声 → 大涟漪
安静 → 无显示
```

**行为影响**：听到声音时 AI 产生 `sound.detected` fact（方向模糊，confidence 低）。AI 可能停下倾听、可能忽略、可能调查。

## 2.4 震动/温度 `Long-term`

后期扩展的感知模态。

# 3. 每种感知必须有独立行为影响

感知不是"揭开地图"。每种感知必须产生**不同的 AI 反应**：

| 感知 | AI fact | 典型反应 |
|------|---------|----------|
| 视觉看到敌人 | `enemy.visible` confidence 1.0 | 根据性格决定打/跑/绕 |
| 嗅觉闻到威胁 | `threat.smell` confidence 0.5-0.7 | 可能绕路、可能调查、可能忽略（取决于好奇心） |
| 听觉听到声音 | `sound.detected` confidence 0.3-0.5 | 可能停下、可能继续、可能紧张 |

**好奇心高的生物**闻到威胁会调查。**谨慎的生物**闻到威胁会绕路。**同一感知在不同性格上产生不同行为。**

---

# 4. 犹豫的表现

当 AI 内部矛盾时（最高分和第二分差距小），玩家必须**立刻感知到**。

## 4.1 视觉表现

```
生物头顶冒出 "？"
生物停下来（或减速）
画面轻微放慢（10-20%）
```

## 4.2 Brain View 叠加

犹豫时 Brain View 自动浮现：

```
它在犹豫
→ 打（勇气核心想打）
→ 跑（嗅觉闻到了威胁）
```

## 4.3 未发生的选择也要高亮

```
它没拿那个资源 — 勇气核心让它优先打敌人
它没走那条路 — 嗅觉闻到了危险
```

**"没做某事"的高亮比"做了某事"更能让玩家理解 AI。**

---

# 5. 信息密度要求

**每 5-10 秒玩家必须从生物那里得到一条新信息。**

信息来源：
- 传感器揭示新区域
- 生物停下/转向/加速/减速
- 嗅觉/听觉检测到东西
- 遇到资源/敌人/危险
- 犹豫时刻
- 性格高亮

**如果连续 10 秒没有任何新信息，节奏就死了。** 地图设计必须保证信息密度。

---

# 6. 地图 = 信息节拍编排

地图不是"关卡"，是"信息节拍序列"。

```
0-5秒：出发，视觉揭开前方，发现分叉
5-10秒：选择一条路，嗅觉检测到某个方向有气味
10-15秒：继续走，视觉发现宝箱
15-20秒：犹豫 — 宝箱旁边有未知危险（嗅觉警告）
20-25秒：你发信号"别去" — 它绕开了
25-30秒：另一条路遇到敌人 — 短暂战斗
30-35秒：战斗结束，继续探索
35-40秒：找到任务资源，返回
```

**40 秒内至少 7 个信息事件。** 地图设计就是编排这些事件的节奏。
