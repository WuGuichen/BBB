# SENSORY DESIGN — 感知与未知设计规范

> 版本 0.9 | 2026-05-27
>
> 这是项目的核心设计文档。"通过他者的感知理解未知"是游戏的核心动词，感知系统的设计质量决定项目成败。
>
> 冲突裁决上：本文档高于 GDD，低于 TECHNICAL/MILESTONES。

---

# 1. 三层信息模型

| 层 | 内容 | 玩家可见？ |
|---|------|-----------|
| **真相层** | 地图实际有什么（敌人/资源/危险/宝箱） | ✗ 编辑器视图 |
| **生物认知层** | 生物通过传感器知道什么 | 部分可见（通过传感器揭示） |
| **玩家认知层** | 玩家看到什么 | 可见（通过生物的眼睛 + 已揭开区域 + 侦察道具） |

**三层的"差异"就是张力的来源**：

| 场景 | 真相 | 生物知道 | 玩家知道 | 体验 |
|------|------|----------|----------|------|
| 双盲 | 敌人在前方 | 不知道 | 不知道 | 紧张（"前面有东西吗？"） |
| 侦察探针 | 敌人在前方 | 不知道 | 知道 | 焦虑（"它要踩雷了！"） |
| 嗅觉检测 | 敌人在前方 | 闻到了 | 闻到了 | 犹豫（"有东西，但不知道在哪"） |
| 视觉发现 | 敌人在前方 | 看到了 | 看到了 | 决断（"打还是跑？"） |

## 1.1 已揭开区域的"记忆衰减"

`MVP`

| 状态 | 视觉 |
|------|------|
| 当前可见（生物视野内） | 全亮，颜色饱和 |
| 曾经可见但当前不在视野（"记忆"） | 颜色冷掉 60%，**动态实体不更新位置** |
| 嗅觉感知到但未视觉揭开 | 颜色暗 + 气味流动粒子可见 |
| 听觉感知到（Post-MVP） | 颜色暗 + 同心圆涟漪 |
| 完全未知 | 黑/雾 |

**"曾经可见但信息可能过期"是这游戏最关键的视觉张力**——你以为那个敌人还在原地，结果它已经摸过来了。

# 2. 感知模态

## 2.1 视觉 `MVP`

生物的"眼"。最基础的感知。

**揭示方式**：以生物为中心，圆形范围揭开迷雾。看到 = 知道位置、类型、状态。

**视觉语言**：实景揭开。迷雾消失，物体显现。

**局限**：黑暗区域视觉范围缩小或无效。被遮挡的物体看不到。

## 2.2 嗅觉 `MVP`

生物的"鼻"。能发现看不到的东西。

**揭示方式**：显示**气味流向**（彩色气流/箭头指向气味来源），但**不显示具体位置和形状**。

**视觉语言**：
```
远处有威胁 → 红色气流从某个方向飘来（连续飘动，强度由透明度表示）
附近有资源 → 绿色气流从某个方向飘来
没有气味 → 无显示
```

**关键差异**：视觉说"那里有一只敌人"，嗅觉说"某个方向有威胁"。**信息精度不同。**

**精度规范**：嗅觉方向有 ±30° 的角度误差，距离信息只有"远 / 中 / 近"三档，不给精确距离。

**行为影响**：嗅觉检测到威胁时，AI 产生 `threat.smell` fact（位置不精确，confidence 0.5-0.7）。AI 可能选择绕路，也可能选择调查（取决于性格）。

## 2.3 听觉 `Post-MVP`

生物的"耳"。能听到声音但不知道在哪。

**揭示方式**：显示**声波涟漪**（从某个大致方向传来），有音量大小但**无精确方位**。

**视觉语言**：
```
远处有脚步声 → 小涟漪
近处有碰撞声 → 大涟漪
安静 → 无显示
```

**行为影响**：听到声音时 AI 产生 `sound.detected` fact（方向模糊，confidence 0.3-0.5）。AI 可能停下倾听、可能忽略、可能调查。

## 2.4 震动/温度 `Long-term`

后期扩展的感知模态。

# 3. 每种感知必须有独立行为影响

感知不是"揭开地图"。每种感知必须产生**不同的 AI 反应**：

| 感知 | AI fact | confidence | 典型反应 |
|------|---------|-----------|----------|
| 视觉看到敌人 | `enemy.visible` | 1.0 | 根据性格决定打/跑/绕 |
| 嗅觉闻到威胁 | `threat.smell` | 0.5-0.7 | 可能绕路、可能调查、可能忽略 |
| 听觉听到声音 | `sound.detected` | 0.3-0.5 | 可能停下、可能继续、可能紧张 |

**好奇心高的生物**闻到威胁会调查。**谨慎的生物**闻到威胁会绕路。**同一感知在不同性格上产生不同行为。**

# 4. AI 反馈四层

> 这是 v0.9 的核心收口。玩家"看懂生物在想什么"不是一件事，是四件事。每一层都必须能独立工作，否则玩家会把"AI 性格"误读成"AI 寻路失败"。

## 4.1 行为层 `MVP` —— 第一眼看懂状态

生物的物理动作就是它当前状态的可读表达。**无 UI 也能读出来。**

| 状态 | 物理表现 |
|------|---------|
| 探索 | 匀速前进，朝向有微小扫动 |
| 警戒 | 减速 50%，朝向锁定威胁方向 |
| 犹豫 | 停下，原地小幅左右晃动 |
| 冲刺 | 加速 30%，朝向锁定 |
| 后退 | 倒退或反向小跑，朝向仍对威胁 |
| 拾取 | 停下，弯腰动作 0.5 秒 |

**占位阶段实现**：圆柱体 + 朝向箭头 + 速度变化。**不需要美术也能做出 6 种可读状态**。

## 4.2 情绪层 `MVP` —— 看懂它紧张/兴奋/害怕

生物头顶图标 + 身体姿态 + 简短音效，传达**情绪而非内容**。

| 情绪 | 头顶 | 姿态 | 音效 |
|------|------|------|------|
| 好奇 | `?` 缓慢飘出 | 头微抬 | 轻"嗯？" |
| 紧张 | `!` 闪烁 | 身体压低 | 短促吸气 |
| 害怕 | `!!` + 抖动 | 身体颤抖 | 急促喘息 |
| 兴奋 | `★` | 身体上扬 | 短促叫声 |
| 不耐烦（低信任时） | `?!` 旋转 | 朝玩家方向偏头 | 哼声 |

**情绪图标必须早于行为变化 0.3-0.5 秒出现**——这是"预兆"，让玩家有反应时间。

## 4.3 理由层 `MVP` —— 看懂内部冲突

屏幕底部行为日志（一行短句，每 5-10 秒最多一条，不刷屏）：

```
0:12  它选择了进攻（勇气核心）
0:18  它没拿那个资源（勇气核心让它优先打敌人）
0:25  它在犹豫（勇气想打，谨慎想跑）
```

**关键设计**：
- **"做了什么"和"没做什么"同等重要**。"没拿那个资源"比"拿了这个资源"更能解释 AI。
- **必须归因到具体零件**："勇气核心"、"嗅觉腔"、"短期记忆"——不要写"AI 判断"这种空话。
- 一行不超过 16 字（中文）。超过就改。

**犹豫时刻的 Brain View 浮现**：

```
它在犹豫
→ 打（勇气核心 +40）
→ 跑（嗅觉闻到威胁 +30）
```

只显示两个候选 + 主导零件，不显示完整 Utility 数学。

## 4.4 结果层 `MVP` —— 看懂关系记忆

玩家信号的反馈，区分四种情况：

| 情况 | 表现 |
|------|------|
| 它听了你的信号并活下来 | "它点了点头"（一次） + 下次犹豫时 Trust 表现层 +1 |
| 它听了你的信号但受伤 | "它迟疑地照做了" |
| 它没听你的信号 | 头顶 `✗` + "它没听你的"（含主导零件原因） |
| 你的信号坑了它 | "它迟疑地看了你一眼" + 下次 Trust 表现层 -1 |

**Trust 表现层 v0**：MVP 阶段实现一个内部 0.0-1.0 变量，只驱动**三处变化**：
1. 犹豫时刻头顶图标变体（高信任 = 温和 `?`，低信任 = 不耐烦 `?!`）
2. 信号反馈短语的选择（如上表）
3. Run 结束总结里一句变体

完整信任度系统（影响 AI Goal priority）在 M6 才做。但**表现层在 M3 就要可见**——没有它，"陪伴"幻想立不起来。

# 5. 犹豫的表现规范

当 AI 内部矛盾时（最高分和第二分差距 < 阈值），玩家必须**立刻感知到**。

## 5.1 视觉表现

```
0.0s  生物头顶 "?" 图标淡入（情绪层先到）
0.2s  生物减速至停止（行为层）
0.3s  Brain View 浮现两个候选（理由层）
0.3s  画面整体慢放至 0.7x，相机微推 1.1x
窗口  3 秒。期间玩家可发信号干预。
3.0s+ 若无干预，自动选最高分，恢复速度
```

## 5.2 触发条件

```
最高 Utility 与次高 Utility 差距 < 15
且 最高分 > 30（否则只是"无事可做"，不是"犹豫"）
且 距离上次犹豫 > 5 秒（避免连续触发）
```

## 5.3 反向高亮（未发生的选择）

犹豫结束后，**显示落选的那个选项 + 落选原因**：

```
它选择了打 — 因为勇气核心 +40 压过谨慎核心 +30
它没选择跑 — 谨慎核心被勇气核心的怒气压制
```

**"它本可以做什么但没做"是这个游戏最有力的认知反馈**。

# 6. 五个核心瞬间

> 这 5 个瞬间撑起整个游戏。如果只在 MVP 做 5 件事，就是这 5 个。**维度、相机、地图都是为这 5 个瞬间服务的。**

## 6.1 ① "它停下来了"——犹豫瞬间 `M4`

**触发**：最高决策与次高决策分差 <15 时。

**表现时间线**：
```
0.0s  生物头顶 "?" 图标淡入（情绪层先到）
0.2s  生物减速至停止（行为层）
0.3s  头顶气泡浮出 2 个候选行动的卡通图标（攻 / 跑）
0.3s  画面整体慢放至 0.7x，相机微推 1.1x
窗口  3 秒。期间玩家可发信号干预。
3.0s+ 若无干预，自动选最高分，恢复速度
```

**为什么重要**：这是这个游戏的标志性瞬间。Pikmin 4 用过类似机制（Pikmin 在指令冲突时短暂迟疑），Frostpunk 也用过（建筑工人罢工前的冰冻瞬间）。**它把"AI 决策"变成"玩家可感知的时刻"。**

**验收**：3 个非开发者看到 "?" 时能说出"它在想什么"。

## 6.2 ② "感官先于视觉"——预警瞬间 `M1`

**触发**：嗅觉腔生物走进未知区域，墙的另一边有敌人/资源。

**表现**：
- 气味流粒子从墙缝飘出，朝向气味源方向
- 生物头顶亮起鼻子图标 + "！"
- 玩家看到了气味（知道有东西），但**不知道是什么**
- 视觉揭开是几秒后的事

**为什么重要**：信息层级在这里被玩家直接体验到。**这是"通过他者的感官"的具象化。** 视觉说"那里有一只敌人"，嗅觉说"某个方向有威胁"——精度不同，但嗅觉先到。

**验收**：3 个非开发者能说出"它闻到了什么，但不知道是什么"。

## 6.3 ③ "性格决断"——选择瞬间 `M2`

**触发**：同一画面内存在多个目标（左边资源、右边敌人）。

**表现**：
- 勇气核心：转向敌人，屏幕底部打出"它选择了进攻（勇气核心）"
- 谨慎核心：转向反方向，"它选择了撤退（谨慎核心）"
- 贪欲核心：转向资源，"它选择了拾取（贪欲核心）"

**为什么重要**：这是"性格读得出来"的验证瞬间。每次都很短（2-3 秒），但每次都让玩家点头："我就知道它会这样"。**如果 playtest 玩家觉得"它就是随机在走"，性格系统失败。**

**验收**：3 个非开发者能说出"这只生物的性格是 XX"。

## 6.4 ④ "信任检验"——干预瞬间 `M5`

**触发**：玩家发出信号（建议/警告/召回）。

**表现**：
- 勇气核心当前在追敌人 → 不听，头顶打出"✗"（反对图标）
- 谨慎核心当前在犹豫 → 听了，朝向调整 + 头顶"✓"
- 结果层反馈："它点了点头" / "它没听你的（勇气核心抵抗）"

**为什么重要**：信号不是按钮，是对话。这一刻玩家感受到"我和它的关系不是完全顺从"——这是同伴感的来源。**如果信号永远被执行，那就是按钮；如果信号有时被拒绝，那就是对话。**

**验收**：3 个非开发者能说出"它有时候听我的，有时候不听"。

## 6.5 ⑤ "理解的爽点"——回顾瞬间 `M6`

**触发**：任务结束（成功 or 失败）。

**表现**：Mission Report 不是表格，是**生物视角的回放叙事**：

```
[回放 1] 0:08  我嗅到了气味，但你没有让我去查
[回放 2] 0:23  我在分叉点犹豫了，因为...
[回放 3] 0:41  我冲向了敌人，因为你装了勇气核心
[回放 4] 0:58  我被偷袭，因为我没看到左侧的洞
[结论]    下次：装嗅觉腔，或选谨慎核心
```

**为什么重要**：这是 L4 认知循环的输入端。它把这次的体验变成下次的假设。**没有它，这游戏就是 5 小时游戏；有了它，这游戏是 30 小时游戏。**

**验收**：3 个非开发者看完 Report 后能说出"下次我要试试不同的配置"。

## 6.6 五个瞬间与里程碑映射

| 瞬间 | 描述 | 首次验证 | 完整验证 |
|------|------|---------|---------|
| ① 犹豫瞬间 | "?" + 慢放 + 相机推近 | M4 | M8 |
| ② 感官先于视觉 | 嗅觉预警，气味流粒子 | M1 | M3 |
| ③ 性格决断 | 同一场景不同性格做不同选择 | M2 | M8 |
| ④ 信任检验 | 信号被部分执行/拒绝 | M5 | M7 |
| ⑤ 理解的爽点 | Mission Report 回放叙事 | M6 | M8 |

**5 个瞬间集齐 = MVP 立项成功。集不齐 = 该停就停。** 这比"完成多少功能"是更准确的成功度量。

# 7. 信息密度要求

**每 5-10 秒玩家必须从生物那里得到一条新信息。**

信息来源（按强度从低到高）：

| 信息类型 | 强度 | 例 |
|---------|------|----|
| 朝向变化 | 弱 | 生物转头扫描 |
| 揭开新区域 | 中 | 视觉范围推进 |
| 感知触发 | 强 | 嗅觉气流出现 |
| 状态切换 | 强 | 探索 → 警戒 |
| 犹豫时刻 | 极强 | "?" 出现 |
| 性格高亮 | 极强 | 底部日志 |
| 战斗 | 极强 | 接触 + 伤害 |

**地图设计目标**：每 60 秒内至少有 7 个"中/强/极强"信息事件。**连续 10 秒没有任何中等以上信息，节奏就死了**。

# 8. M1 验收剧本：60 秒首房脚本

> 这不是"附加文档"，是 M1 的验收剧本。M1 完成的标准是：能跑通这个 60 秒，且 3 个非开发者看完能说出脚本里"玩家应该感受到"那一栏的内容。

## 7.1 房间布局

```
            ┌─────────出口──────────┐
            │                        │
            │  [资源点]   [威胁敌人]  │
            │                        │
            ├──墙──┐         ┌──墙──┤
            │       │ 分叉    │       │
            │       └─┐    ┌─┘       │
            │         │    │         │
            │   [起点：生物]           │
            └─────────────────────────┘
```

- 大小：20×20m，被矮墙分成 3 个子区域
- 起点：底部中央
- 分叉点：起点正前方，左右两条路
- 左路：通向资源点（视觉揭开易）
- 右路：通向威胁（嗅觉先于视觉触发）
- 出口：顶部中央

## 7.2 配置

- 生物：勇气核心 + 嗅觉腔
- 干预道具：**警告 ×2，建议 ×1**（不测稀缺性，测沟通感）
- 目标：拿到资源点的资源 + 走到出口

## 7.3 60 秒节拍脚本

| 时间 | 世界事件 | 生物反应（行为/情绪） | 玩家信息（理由/结果） | 玩家可干预 |
|------|---------|---------------------|---------------------|----------|
| 0-5s | 投放，前方视野约 5m | 自动前进，朝向箭头 | "出发了" | 无 |
| 5-10s | 走到分叉点 | 减速 + 朝向左右扫动 | 行为层"扫描"可读 | 无 |
| 10-15s | 嗅觉触发：右侧威胁气味 | 头顶 `!`，身体压低 | 红色气流从右飘来 | 无 |
| 15-20s | 看到左路尽头资源轮廓 | 转向左路 | 底部日志："它选择了拿资源（勇气核心）" | 无 |
| 20-25s | 接近资源 | 加速冲刺 | 行为层加速可读 | 无 |
| 25-30s | 拾取资源 | 停下，弯腰 0.5s | 底部日志："拾取" | 无 |
| 30-35s | 朝出口方向，但前方有未揭开区域 | 停下，朝向两边轻晃 | 头顶 `?`，慢放 0.7x | **建议 → 直走** |
| 35-40s | 视觉揭开：通往出口的走廊有威胁残影（嗅觉早于视觉） | 转向 + `!` | 红色气流出现，行为转警戒 | **警告 → 它绕开** |
| 40-45s | 绕开威胁的另一条路 | 加速前进 | 底部日志："它听了你的警告（点了点头）" | 无 |
| 45-55s | 到达出口区域，沿途短暂战斗（一个弱敌） | 3 秒接触 + 死亡敌人 | 底部日志：性格主导战斗 | 无（演示战斗是事件） |
| 55-60s | 走出出口 | 朝出口前进 | Run 结束 → Mission Report 简版 | 无 |

## 7.4 玩家应该感受到（验收）

3 个非开发者看完 60 秒后，**应该能复述至少 3 句**：

- "它在分叉那里好像在想东西"
- "它闻到了右边有东西"
- "我喊它别去，它绕开了"
- "它好像点头了"
- "战斗很快，不像主菜"
- "我想知道下一个房间长什么样"

**核心问题（必须问）**：

> "你愿不愿意再派同一只生物出去？"

如果答"愿意" → 陪伴感成立。
如果答"无所谓换一只" → 角色化（命名/伤痕）不够，回看 GDD §13。
如果答"不愿意，它太蠢了" → AI 可读性不够，回看 §4 四层反馈。

## 7.5 不通过的处理

| Playtest 反馈 | 原因 | 应对 |
|-------------|------|------|
| "它在随便走" | 行为层不可读 | 加大状态切换的速度差，加朝向箭头 |
| "我不知道它闻到了什么" | 嗅觉视觉语言失败 | 加大气流色彩对比，加方向箭头 |
| "我喊它它好像没反应" | 结果层缺失 | 加"它点了点头"短语 + 头顶图标 |
| "我看不懂它为什么这样" | 理由层失败 | 改底部日志措辞，归因到具体零件 |

**M1 不通过就不要走 M2。** 这不是"加功能"问题，是核心动词是否成立的问题。

# 8.5 60 秒理想投放（完整体验示例）

> 这是整个设计的内聚检验。它验证维度+感知+地图+瞬间+干预是否自洽。不同于 §8.3 的 M1 验收剧本（简化版），这里展示**完整体验**。

```
0:00  组装完成：勇气核心 + 嗅觉腔 + 厚甲
      携带信号：召回信标 ×1，警告标记 ×1
0:02  投放到房间。视野只有 5m 半径
0:05  生物开始向前走，头顶箭头朝前
0:08  ★ 嗅觉触发：左前方传来气味流粒子
      生物头顶鼻子图标亮起
      → 我（玩家）犹豫：是气味强源（资源）还是敌人？
0:12  生物自主转向左前，进入走廊
0:18  视觉揭开：左前方有一个宝箱，旁边有抓痕
      行为日志："它向资源靠近"（贪欲？不，勇气核心也会拿）
0:23  ★ 分叉点：宝箱在左，正前方传来更深的气味
      生物停下，"?"图标
      慢放，相机推近
      → 我决策：留着信号还是用？
      → 我用"询问"：揭开它当前知道的——前方气味是"血腥"
      → 这意味着前方有受伤的敌人 = 可能可击杀
0:28  我（无信号干预）让它自决
      勇气核心 → 选择前方
      行为日志："它选择了进攻（勇气核心）"
0:35  视觉揭开：是一只 mid-tier 敌人，HP 已减半
      接触战斗，相机微推 + 慢放
0:38  生物贴脸开打，伤害互换
0:42  敌人死亡，掉落零件
0:45  生物继续向前。HP 剩 60%
0:50  ★ 新房间：宽阔，多敌人，右侧有出口
      勇气核心想冲，但我看 HP 60% 觉得风险大
      → 我用"召回信标"
      行为日志："它收到召回，调整方向（勇气核心抵抗中）"
      → 生物部分听话，向出口移动但绕路打路上的敌人
0:58  到达出口
      行为日志："它带回了 1 个零件"
1:00  Mission Report：
        - 成功
        - 嗅觉腔提前预警 1 次
        - 犹豫 1 次（你帮它做了决定）
        - 召回信号被部分执行（勇气核心抵抗）
        - 推荐：下次配厚甲撑住勇气流，或换谨慎核心降低风险
```

**这 60 秒里玩家做了什么？**
- 观察 7-8 次生物状态变化
- 用了 2 个信号（询问 + 召回）
- 做了 3 个决策（让它自决/用信号/什么时候召回）
- 学到了 2-3 件事（嗅觉预警 / 勇气核心不完全听召回 / 残血进新房间风险大）

**信息更新密度 = ~10 个/分钟。** 远超"5-10 秒一个事件"的目标。这就是"不无聊"的真正答案。

**5 个核心瞬间在这个示例中的分布**：
- ② 感官先于视觉：0:08 嗅觉触发
- ③ 性格决断：0:28 勇气核心选择进攻
- ① 犹豫瞬间：0:23 分叉点 "?" + 慢放
- ④ 信任检验：0:50 召回信标被部分执行
- ⑤ 理解的爽点：1:00 Mission Report

# 9. 地图 = 信息节拍编排

地图不是"关卡"，是"信息节拍序列"。每张房间必须按 §7 的密度要求设计：

```
每 5-10 秒一个"中/强/极强"信息事件
40 秒内至少 7 个事件
60 秒内至少 10 个事件
```

参见 §8 的 60 秒脚本作为节拍编排示范。

## 9.1 必备节拍元素

每张地图至少包含 **5 种**：

| 节拍类型 | 触发反应 | 玩家学到 |
|---------|--------|---------|
| 视野分叉 | 减速 + 扫描 | 性格如何决断 |
| 气味源 | 嗅觉提前感知 | 嗅觉早于视觉 |
| 视觉阻挡转角 | 减速 + 朝向锁定 | 信息层差 |
| 诱饵（看似资源实为陷阱） | 贪欲生物上当 | 性格的代价 |
| 痕迹（脚印/血迹） | 短期记忆触发 | 环境叙事 |
| 资源点 | 拾取行为 | 优先级取舍 |
| 危险地形 | 抗性决定通过 | 身体决定能力 |
| 狭窄通道 | 体型约束 | 身体的约束 |

## 9.2 制作方式

MVP 阶段：**手工设计 10 个房间模板**，每个有 2-4 个连接口；程序化拼接成 Run。

具体模板规范、节拍元素清单、拼接规则、密度自检规范见 [Map_Templates.md](Map_Templates.md)。

## 9.3 与 §8 的关系

§8 的 60 秒验收剧本对应 [Map_Templates R01-Lab](Map_Templates.md#41-r01-lab--初心房m1-验收剧本专用)。

R01 是 M1 唯一房间，节拍元素配置见 Map_Templates §4.1。
