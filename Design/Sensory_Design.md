# SENSORY DESIGN — 感知与未知设计规范

> 版本 2.0 | 2026-05-28
>
> 这是项目的核心设计文档之一。玩家通过生物的感官理解未知世界，感知系统的设计质量决定"观察"这个核心动词是否成立。
>
> 冲突裁决上：本文档高于 GDD，低于 TECHNICAL/MILESTONES。

---

# 1. 三层信息模型

| 层 | 内容 | 玩家可见？ |
|---|------|-----------|
| **真相层** | 地图实际有什么（敌人/资源/危险/宝箱） | ✗ 编辑器视图 |
| **生物认知层** | 生物通过传感器知道什么 | 部分可见（通过传感器揭示） |
| **玩家认知层** | 玩家看到什么 | 可见（通过生物的眼睛 + 已揭开区域 + 侦察道具） |

**三层的"差异"就是张力的来源**：

| 场景 | 真相 | 生物知道 | 玩家知道 | 体验 |
|------|------|----------|----------|------|
| 双盲 | 敌人在前方 | 不知道 | 不知道 | 紧张（"前面有东西吗？"） |
| 侦察探针 | 敌人在前方 | 不知道 | 知道 | 焦虑（"它要踩雷了！"） |
| 嗅觉检测 | 敌人在前方 | 闻到了 | 闻到了 | 犹豫（"有东西，但不知道在哪"） |
| 视觉发现 | 敌人在前方 | 看到了 | 看到了 | 决断（"打还是跑？"） |

## 1.1 已揭开区域的"记忆衰减"

`MVP`

| 状态 | 视觉 |
|------|------|
| 当前可见（生物视野内） | 全亮，颜色饱和 |
| 曾经可见但当前不在视野（"记忆"） | 颜色冷掉 60%，**动态实体不更新位置** |
| 嗅觉感知到但未视觉揭开 | 颜色暗 + 气味流动粒子可见 |
| 听觉感知到（Post-MVP） | 颜色暗 + 同心圆涟漪 |
| 完全未知 | 黑/雾 |

**"曾经可见但信息可能过期"是这游戏最关键的视觉张力**——你以为那个敌人还在原地，结果它已经摸过来了。

# 2. 感知模态

## 2.1 视觉 `MVP`

生物的"眼"。最基础的感知。

**揭示方式**：以生物为中心，圆形范围揭开迷雾。看到 = 知道位置、类型、状态。

**视觉语言**：实景揭开。迷雾消失，物体显现。

**局限**：黑暗区域视觉范围缩小或无效。被遮挡的物体看不到。

## 2.2 嗅觉 `MVP`

生物的"鼻"。能发现看不到的东西。

**揭示方式**：显示**气味流向**（彩色气流/箭头指向气味来源），但**不显示具体位置和形状**。

**视觉语言**：
```
远处有威胁 → 红色气流从某个方向飘来（连续飘动，强度由透明度表示）
附近有资源 → 绿色气流从某个方向飘来
没有气味 → 无显示
```

**关键差异**：视觉说"那里有一只敌人"，嗅觉说"某个方向有威胁"。**信息精度不同。**

**精度规范**：嗅觉方向有 ±30° 的角度误差，距离信息只有"远 / 中 / 近"三档，不给精确距离。

**行为影响**：嗅觉检测到威胁时，AI 产生 `threat.smell` fact（位置不精确，confidence 0.5-0.7）。AI 可能选择绕路，也可能选择调查（取决于性格）。

## 2.3 听觉 `Post-MVP`

生物的"耳"。能听到声音但不知道在哪。

**揭示方式**：显示**声波涟漪**（从某个大致方向传来），有音量大小但**无精确方位**。

**视觉语言**：
```
远处有脚步声 → 小涟漪
近处有碰撞声 → 大涟漪
安静 → 无显示
```

**行为影响**：听到声音时 AI 产生 `sound.detected` fact（方向模糊，confidence 0.3-0.5）。AI 可能停下倾听、可能忽略、可能调查。

## 2.4 震动/温度 `Long-term`

后期扩展的感知模态。

# 3. 每种感知必须有独立行为影响

感知不是"揭开地图"。每种感知必须产生**不同的 AI 反应**：

| 感知 | AI fact | confidence | 典型反应 |
|------|---------|-----------|----------|
| 视觉看到敌人 | `enemy.visible` | 1.0 | 根据性格决定打/跑/绕 |
| 嗅觉闻到威胁 | `threat.smell` | 0.5-0.7 | 可能绕路、可能调查、可能忽略 |
| 听觉听到声音 | `sound.detected` | 0.3-0.5 | 可能停下、可能继续、可能紧张 |

**好奇心高的生物**闻到威胁会调查。**谨慎的生物**闻到威胁会绕路。**同一感知在不同性格上产生不同行为。**

# 4. AI 反馈四层

> 这是 v0.9 的核心收口。玩家"看懂生物在想什么"不是一件事，是四件事。每一层都必须能独立工作，否则玩家会把"AI 性格"误读成"AI 寻路失败"。

## 4.1 行为层 `MVP` —— 第一眼看懂状态

生物的物理动作就是它当前状态的可读表达。**无 UI 也能读出来。**

| 状态 | 物理表现 |
|------|---------|
| 探索 | 匀速前进，朝向有微小扫动 |
| 警戒 | 减速 50%，朝向锁定威胁方向 |
| 犹豫 | 停下，原地小幅左右晃动 |
| 冲刺 | 加速 30%，朝向锁定 |
| 后退 | 倒退或反向小跑，朝向仍对威胁 |
| 拾取 | 停下，弯腰动作 0.5 秒 |

**占位阶段实现**：圆柱体 + 朝向箭头 + 速度变化。**不需要美术也能做出 6 种可读状态**。

## 4.2 情绪层 `MVP` —— 看懂它紧张/兴奋/害怕

生物头顶图标 + 身体姿态 + 简短音效，传达**情绪而非内容**。

| 情绪 | 头顶 | 姿态 | 音效 |
|------|------|------|------|
| 好奇 | `?` 缓慢飘出 | 头微抬 | 轻"嗯？" |
| 紧张 | `!` 闪烁 | 身体压低 | 短促吸气 |
| 害怕 | `!!` + 抖动 | 身体颤抖 | 急促喘息 |
| 兴奋 | `★` | 身体上扬 | 短促叫声 |
| 不耐烦（低信任时） | `?!` 旋转 | 朝玩家方向偏头 | 哼声 |

**情绪图标必须早于行为变化 0.3-0.5 秒出现**——这是"预兆"，让玩家有反应时间。

## 4.3 理由层 `MVP` —— 看懂内部冲突

屏幕底部行为日志（一行短句，每 5-10 秒最多一条，不刷屏）：

```
0:12  它选择了进攻（勇气核心）
0:18  它没拿那个资源（勇气核心让它优先打敌人）
0:25  它在犹豫（勇气想打，谨慎想跑）
```

**关键设计**：
- **"做了什么"和"没做什么"同等重要**。"没拿那个资源"比"拿了这个资源"更能解释 AI。
- **必须归因到具体零件**："勇气核心"、"嗅觉腔"、"短期记忆"——不要写"AI 判断"这种空话。
- 一行不超过 16 字（中文）。超过就改。

**犹豫时刻的 Brain View 浮现**：

```
它在犹豫
→ 打（勇气核心 +40）
→ 跑（嗅觉闻到威胁 +30）
```

只显示两个候选 + 主导零件，不显示完整 Utility 数学。

## 4.4 结果层 `MVP` —— 看懂关系记忆

玩家信号的反馈，区分四种情况：

| 情况 | 表现 |
|------|------|
| 它听了你的信号并活下来 | "它点了点头"（一次） + 下次犹豫时 Trust 表现层 +1 |
| 它听了你的信号但受伤 | "它迟疑地照做了" |
| 它没听你的信号 | 头顶 `✗` + "它没听你的"（含主导零件原因） |
| 你的信号坑了它 | "它迟疑地看了你一眼" + 下次 Trust 表现层 -1 |

**Trust 表现层 v0**：MVP 阶段实现一个内部 0.0-1.0 变量，只驱动**三处变化**：
1. 犹豫时刻头顶图标变体（高信任 = 温和 `?`，低信任 = 不耐烦 `?!`）
2. 信号反馈短语的选择（如上表）
3. Run 结束总结里一句变体

完整信任度系统（影响 AI Goal priority）在 M6 才做。但**表现层在 M3 就要可见**——没有它，"陪伴"幻想立不起来。

# 5. 犹豫的表现规范

当 AI 内部矛盾时（最高分和第二分差距 < 阈值），玩家必须**立刻感知到**。

## 5.1 视觉表现

```
0.0s  生物头顶 "?" 图标淡入（情绪层先到）
0.2s  生物减速至停止（行为层）
0.3s  Brain View 浮现两个候选（理由层）
0.3s  画面整体慢放至 0.7x，相机微推 1.1x
窗口  3 秒。期间玩家可发信号干预。
3.0s+ 若无干预，自动选最高分，恢复速度
```

## 5.2 触发条件

```
最高 Utility 与次高 Utility 差距 < 15
且 最高分 > 30（否则只是"无事可做"，不是"犹豫"）
且 距离上次犹豫 > 5 秒（避免连续触发）
```

## 5.3 反向高亮（未发生的选择）

犹豫结束后，**显示落选的那个选项 + 落选原因**：

```
它选择了打 — 因为勇气核心 +40 压过谨慎核心 +30
它没选择跑 — 谨慎核心被勇气核心的怒气压制
```

**"它本可以做什么但没做"是这个游戏最有力的认知反馈**。

# 6. 五个核心瞬间

> 这 5 个瞬间撑起整个游戏。如果只在 MVP 做 5 件事，就是这 5 个。**维度、相机、地图都是为这 5 个瞬间服务的。**

## 6.1 ① "它停下来了"——犹豫瞬间 `M4`

**触发**：最高决策与次高决策分差 <15 时。

**表现时间线**：
```
0.0s  生物头顶 "?" 图标淡入（情绪层先到）
0.2s  生物减速至停止（行为层）
0.3s  头顶气泡浮出 2 个候选行动的卡通图标（攻 / 跑）
0.3s  画面整体慢放至 0.7x，相机微推 1.1x
窗口  3 秒。期间玩家可发信号干预。
3.0s+ 若无干预，自动选最高分，恢复速度
```

**为什么重要**：这是这个游戏的标志性瞬间。Pikmin 4 用过类似机制（Pikmin 在指令冲突时短暂迟疑），Frostpunk 也用过（建筑工人罢工前的冰冻瞬间）。**它把"AI 决策"变成"玩家可感知的时刻"。**

**验收**：3 个非开发者看到 "?" 时能说出"它在想什么"。

## 6.2 ② "感官先于视觉"——预警瞬间 `M1`

**触发**：嗅觉腔生物走进未知区域，墙的另一边有敌人/资源。

**表现**：
- 气味流粒子从墙缝飘出，朝向气味源方向
- 生物头顶亮起鼻子图标 + "！"
- 玩家看到了气味（知道有东西），但**不知道是什么**
- 视觉揭开是几秒后的事

**为什么重要**：信息层级在这里被玩家直接体验到。**这是"通过他者的感官"的具象化。** 视觉说"那里有一只敌人"，嗅觉说"某个方向有威胁"——精度不同，但嗅觉先到。

**验收**：3 个非开发者能说出"它闻到了什么，但不知道是什么"。

## 6.3 ③ "性格决断"——选择瞬间 `M2`

**触发**：同一画面内存在多个目标（左边资源、右边敌人）。

**表现**：
- 勇气核心：转向敌人，屏幕底部打出"它选择了进攻（勇气核心）"
- 谨慎核心：转向反方向，"它选择了撤退（谨慎核心）"
- 贪欲核心：转向资源，"它选择了拾取（贪欲核心）"

**为什么重要**：这是"性格读得出来"的验证瞬间。每次都很短（2-3 秒），但每次都让玩家点头："我就知道它会这样"。**如果 playtest 玩家觉得"它就是随机在走"，性格系统失败。**

**验收**：3 个非开发者能说出"这只生物的性格是 XX"。

## 6.4 ④ "信任检验"——干预瞬间 `M5`

**触发**：玩家发出信号（建议/警告/召回）。

**表现**：
- 勇气核心当前在追敌人 → 不听，头顶打出"✗"（反对图标）
- 谨慎核心当前在犹豫 → 听了，朝向调整 + 头顶"✓"
- 结果层反馈："它点了点头" / "它没听你的（勇气核心抵抗）"

**为什么重要**：信号不是按钮，是对话。这一刻玩家感受到"我和它的关系不是完全顺从"——这是同伴感的来源。**如果信号永远被执行，那就是按钮；如果信号有时被拒绝，那就是对话。**

**验收**：3 个非开发者能说出"它有时候听我的，有时候不听"。

## 6.5 ⑤ "理解的爽点"——回顾瞬间 `M6`

**触发**：任务结束（成功 or 失败）。

**表现**：Mission Report 不是表格，是**生物视角的回放叙事**：

```
[回放 1] 0:08  我嗅到了气味，但你没有让我去查
[回放 2] 0:23  我在分叉点犹豫了，因为...
[回放 3] 0:41  我冲向了敌人，因为你装了勇气核心
[回放 4] 0:58  我被偷袭，因为我没看到左侧的洞
[结论]    下次：装嗅觉腔，或选谨慎核心
```

**为什么重要**：这是 L4 认知循环的输入端。它把这次的体验变成下次的假设。**没有它，这游戏就是 5 小时游戏；有了它，这游戏是 30 小时游戏。**

**验收**：3 个非开发者看完 Report 后能说出"下次我要试试不同的配置"。

## 6.6 五个瞬间与里程碑映射

| 瞬间 | 描述 | 首次验证 | 完整验证 |
|------|------|---------|---------|
| ① 犹豫瞬间 | "?" + 慢放 + 相机推近 | M4 | M8 |
| ② 感官先于视觉 | 嗅觉预警，气味流粒子 | M1 | M3 |
| ③ 性格决断 | 同一场景不同性格做不同选择 | M2 | M8 |
| ④ 信任检验 | 信号被部分执行/拒绝 | M5 | M7 |
| ⑤ 理解的爽点 | Mission Report 回放叙事 | M6 | M8 |

**5 个瞬间集齐 = MVP 立项成功。集不齐 = 该停就停。** 这比"完成多少功能"是更准确的成功度量。

# 7. 信息密度

**密度由系统保证，不是设计师埋点。**

信息来源是房间里活物的自然行为：

| 信息来源 | 强度 | 怎么产生 |
|---------|------|---------|
| 活物移动/转向 | 弱 | 多智能体独立运作的自然结果 |
| 感知触发（气味/声音） | 中 | 环境实体散发的气味，活物行为产生的声音 |
| 活物之间交互 | 强 | 敌人打小生物、清道夫吃尸体、巡逻者被酸池逼走 |
| 玩家生物的状态变化 | 强 | 犹豫/拾取/战斗——内部需求×性格的涌现 |

**设计师管"放多少活物"，不管"什么时候发生什么事"。** 每个房间至少 3-4 个独立 AI 主体（见 [World_Simulation §7](World_Simulation.md#7-活物密度规范)），活物的行为自然产生信息密度。

**不达标的表现**：房间里玩家干瞪眼 → 活物不够，加。

# 8. M1 验收标准

**不再是"跑通 60 秒剧本"，是"同一房间播放 3 次，事件序列必须显著不同"。**

详细验收标准见 [World_Simulation §8](World_Simulation.md#8-m1-验收标准涌现版)。

核心指标：
- 3 次播放中至少出现 5 种不同"主要事件"
- 没有任何 2 次有 80% 相同的事件序列
- 3 个非开发者各看一次，每人说出的故事都不一样
- 玩家看 3 次后能说"我下次想试试 X，看会不会发生 Y"

**M1 的问题不是"做出 60 秒戏"，是"做出一个会产生不同 60 秒的盒子"。**

# 9. 地图 = 活物配置

地图不是"节拍序列"，是**活物和环境规则的容器**。

设计师放活物和环境元素，不管它们什么时候互动：

```
每个房间至少 3-4 个独立 AI 主体
每个环境元素有自己的规则
它们碰巧在空间里交互 → 产生事件
```

参见 [World_Simulation §2-§7](World_Simulation.md) 的多智能体仿真规范和活物密度规范。

## 9.1 房间模板定义

房间模板定义**地形 + 活物配置 + 倾向规则**，不定义节拍序列：

```csharp
public sealed class RoomTemplate
{
    public string TemplateId;
    public RoomCategory Category;
    public Vec2Int Size;
    public List<ConnectionPort> Ports;
    public List<AgentSpawn> Spawns;      // 放什么活物
    public List<EnvEntity> EnvEntities;  // 放什么环境元素
    public List<StaticObstacle> Obstacles;
}
```

具体模板规范见 [Map_Templates.md](Map_Templates.md)。

## 9.2 与 World_Simulation 的关系

§7 的信息密度由活物行为自然产生，不由设计师埋点保证。活物密度规范见 [World_Simulation §7](World_Simulation.md#7-活物密度规范)。
