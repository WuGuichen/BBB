## 一句话

> **设计一个系统，让它自己跑。世界测试你的设计，失败给你数据，改造后再来。**

你不是在"养"生物。你是在**造一台机器**，扔进危险环境，看它能不能活下来。

## 核心问题

**这次能走多远？**

不是"我的生物可不可爱"，是"我设计的系统扛不扛得住"。

## 两条铁律

- **铁律 A — 无暗骰子。** 结果 100% 可追溯到你本可知道的原因。没有隐藏 RNG。
- **铁律 B — 无实体特判。** 没有 `if (对方是清道夫)` 的硬编码。一切交互通过总线的刺激/信念/标签产生。涌现的根。

## 总线架构（System_Core）

```
动作发射刺激 → 环境传播 → 感官过滤成信念 → 信念×需求×性格算出效用 → 产生新动作
```

万物皆发射器+接收器。你改一个标签，因果顺着总线传播，重写所有下游故事。

详见 [System_Core.md](System_Core.md)。

## 玩家身份

**系统工程师。** 不是宠物主人，不是指挥官。

你做的事：
- 选部件（物理层）
- 写规则（行为层）
- 按下启动
- 看它跑
- 读报告
- 改设计
- 再来

你**不在世界里**。你通过你造的机器的传感器看世界。

## 核心循环

```
组装（部件 + AI蓝图）→ 投入生态区 → 系统自主运行 → 撤退或全灭 → 报告 → 改造 → 再来
```

## 设计支柱

### 1. 系统先于内容

总线是脊椎。刺激→信念→效用。所有实体共用一条总线，没有特判。

设计师只摆名词（EmitterDef + SensorDef + 性格权重），换种子就长出不同故事。

### 2. 身体是物理约束

腿 = 移动方式。眼 = 感知通道。爪 = 伤害类型。甲 = 抗性类型。

不是"选可爱外观"，是"选物理能力集"。

### 3. AI 是行为生成器

卡片式规则编译成 PersonalityBias + Relevance，喂进效用公式。

同一个身体 + 不同 AI = 完全不同的机器。

### 4. 世界是测试环境

酸池扩不扩散不在乎你来了没有。巡逻者有自己的目标。清道夫跟着尸体走。

你的机器扔进去，世界不伺候你。

### 5. 信息是武器也是牢笼

视觉给精确位置，嗅觉只给方向±30°。你只能通过机器的传感器看世界。

**错误信念三态**：过期（衰减没更新）/ 错误（把清道夫当威胁）/ 缺失（没有对应传感器）。

这三态是策略的核心战场。

### 6. 死亡是数据

生物死了 = 你的设计有缺陷。报告告诉你哪里有问题。

不是"好可怜"，是"我知道怎么改了"。

## Run 结构

```
3-5 个连续生态区，难度递增。
Run 内部件/信号死了丢。
样本/数据死了保留。
撤退 = 安全但放弃后面奖励。
全灭 = 只剩样本和数据。
```

**核心决策：继续还是撤退？**

## 硬核要素

| 要素 | 设计 |
|------|------|
| 无暗骰子 | 结果可追溯，策略可推理 |
| 无实体特判 | 涌现空间最大化 |
| 信息不完整 | 嗅觉给方向不给答案，视觉被墙阻断 |
| 资源稀缺 | 信号 0-2 个，用完没了 |
| 死亡有代价 | Run 内部件全丢 |
| 撤退是赌博 | 继续 = 高风险高回报 |

## 维度

等距 2.5D，55-60° 相机角，正交投影。
