# CONCEPT — Creature Builder Roguelite

> 版本 2.0 | 2026-05-28

## 一句话定位

设计生物的身体和AI，投放到危险生态区，观察它们自主行动、失败、成功，回收资源继续改造。

> **设计身体，编排行为，投放生态，观察结果，回收改造。**

## 核心循环

```
设计生物（身体部件 + AI蓝图）
↓
投放到生态区（危险环境 + 野生生物）
↓
生物自主行动（按你设计的AI规则）
↓
观察（通过它的感知看世界）
↓
间接干预（有限信号）
↓
结果（成功/失败/死亡）
↓
分析（战后报告 + 数据）
↓
改造（换部件 + 调AI规则）
↓
再测试
```

## 核心卖点

玩家不是直接操控角色，而是**设计一个系统，让它自己运转**。

```
传统RPG：玩家控制角色 → 打怪 → 升级 → 换装备
本游戏：玩家设计生物 → 生物自主行动 → 世界测试设计 → 玩家分析失败 → 重新设计
```

它更接近：
- **Cogmind** 的零件构建
- **Screeps** 的行为逻辑设计
- **Spore** 的身体部件影响能力
- **RimWorld** 的系统生成故事

## 设计支柱

### A. 身体决定能力

生物不是一张皮，而是一组功能部件。

```
腿决定移动方式
眼睛决定感知距离
耳朵决定听觉能力
爪子决定近战能力
甲壳决定防御
脑核决定AI复杂度
```

身体不是纯美术，而是玩法数据。

### B. AI决定性格

同一个身体，用不同AI蓝图会有完全不同的表现。

```
胆小型：远距离观察，残血逃跑
狂暴型：受伤后攻击欲上升，几乎不撤退
拾荒型：避战，优先吃尸体和捡资源
伏击型：潜伏，等猎物靠近后突袭
```

玩家配置行为倾向，调节判断条件，观察AI是否如预期工作。

### C. 世界负责测试设计

每个环境都是一张"考试卷"。

```
酸性沼泽：测试抗性、路径选择
黑暗洞穴：测试听觉、嗅觉
机械废墟：测试潜行、电子干扰
火山地带：测试热抗性、逃跑逻辑
```

玩家不是单纯打赢敌人，而是在问：**我设计的生物能不能适应这个生态位？**

### D. 失败必须有价值

生物死亡不只是惩罚，而是数据。

```
死亡原因：恐惧过高，逃跑状态体力耗尽
建议观察：逃跑阈值过低，缺少安全巢穴识别
回收资源：破损腿部x2，行为数据x36
```

玩家失败后会想：**我知道怎么改了。**

### E. 涌现来自规则自由交互

世界有自己的生命。敌人/野生生物有自己的目标，环境独立运作。

```
敌人不只追你，有自己的目标
酸池独立扩散，不管谁在里面
生物有需求（饥饿/疲劳/恐惧），可能不听你的
```

设计师放活物、调参数、观察生态，不安排剧本。

## 维度：等距 2.5D

| 方案 | 沉浸感 | 感知可视化 | 多生物注意力 | 美术成本 | 占位友好 |
|------|--------|----------|------------|---------|---------|
| 纯2D俯视 | 中 | 优 | 优 | 低 | 优 |
| **2.5D等距** | **良** | **优** | **优** | **中** | **良** |
| 跟随3D | 优 | 中 | 差 | 高 | 差 |

**2.5D等距是甜蜜点**：框架已有AABB天然适配，占位体可读，迷雾/感知用shader处理，多生物同屏可读。

**MVP起步参数**：相机角度55-60°，正交投影，单位高度1m。

> 参照系：**Hades**的视角 + **Pikmin**的多生物 + **Darkest Dungeon**的状态可读性

## 玩家身份

**荒野生物工程师** — 边境实验站的管理员，用半生物半机械的造物探索危险生态区，回收资源，解锁身体部件和AI模块。

## 策略深度来源

| 来源 | 设计 |
|------|------|
| **投放前** | 选什么部件 = 选什么能力 = 选什么风险 |
| **AI配置** | 配什么规则 = 遇到什么情况怎么反应 |
| **投放中** | 2个信号，什么时候用？它可能不听 |
| **信息不完整** | 气味只给方向，你必须赌 |
| **时间压力** | 酸池在扩，巡逻者在转，窗口在关 |
| **死亡有代价** | 生物死了，这次Run结束 |
| **失败有价值** | 数据+回收+改造方向 |

## 硬核要素

| 要素 | 设计 |
|------|------|
| 死亡 | 生物死了就没了，Run结束。尸体留在地图上 |
| 资源稀缺 | 每次投放0-2个信号 |
| 信息不完整 | 感知只给方向+强度，不给答案 |
| 选择有后果 | 走了左边就永远看不到右边 |
| 时间压力 | 环境在变化，窗口在关 |

## 关键风险

| 风险 | 解决 |
|------|------|
| 玩家看不懂AI在干什么 | 强反馈：意图UI + 决策解释 + 战后日志 |
| 不能直接控制很烦 | 有限但强力的间接指挥，不听话有合理原因 |
| 构筑太复杂新手进不来 | 预设模板 + 卡片规则 + 失败后建议 |
| 所有玩家找到最优解 | 环境多样化 + 生态反制 + 资源约束 |
