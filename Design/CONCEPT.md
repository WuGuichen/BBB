# CONCEPT — AI Creature Sandbox

> 版本 0.4 | 2026-05-27

## 一句话定位

玩家不是直接操控角色，而是**设计生物的身体和 AI 行为逻辑**，投放到危险环境中观察它们自主行动，通过分析失败、回收资源来迭代改造。

> **设计身体，编排行为，投放生态，观察结果，回收改造。**

## 核心玩法循环

```
设计生物（身体部件 + AI 蓝图）
    ↓
投放到任务地图
    ↓
生物自主行动（探索/采集/战斗/逃跑）
    ↓
玩家间接干预（信标/信号）
    ↓
任务结束 → 战后报告（行为日志 + 失败原因 + 改进建议）
    ↓
修改设计 → 再次投放
```

## 四大设计支柱

| 支柱 | 一句话 |
|------|--------|
| **身体决定能力** | 部件不是美术，是玩法数据。腿决定移动方式，眼决定感知范围，甲壳决定防御。 |
| **AI 决定性格** | 同一身体，不同 AI 蓝图有完全不同表现。胆小拾荒者 vs 狂暴猎手。 |
| **世界测试设计** | 每个环境是一张考试卷。酸沼测试抗性，暗洞测试听觉，废墟测试潜行。 |
| **失败必须有价值** | 死亡不是惩罚，是信息。尸体、行为日志、AI 失败原因、新改造方向。 |

## 核心风险：涌现 vs 噪声

这个品类最大的陷阱：**当 AI 表现得"还行"时玩家觉得无聊；当 AI 表现得"出乎意料"时玩家不知道是自己设计巧妙还是系统耦合的偶然。**

Rain World 成立是因为每次"惊喜"都能被玩家事后归因。Spore 失败是因为改了身体却感受不到行为差异。

本项目的解法：
- **Brain View 让 AI 决策可归因**（不是显示数字，是自然语言解释）
- **失败分类让死亡可理解**（不是"死了"，是"因为进食优先级过高偏离了任务路径"）
- **部件差异让修改有意义**（换了部件后行为必须可感知地变化）

## 竞品对比

| 游戏 | 相似点 | 不同点 |
|------|--------|--------|
| **Ecosystem** | DNA 生成身体 + 神经系统 + 战斗能力 + 进化 | 水下生态为主；无玩家 AI 蓝图编辑 |
| **Rain World** | 程序化动画 + 捕食者/猎物生态 | 2D；玩家不创造生物/AI |
| **Niche** | 基因适应 + 生态位 + biome 筛选 | 回合制；无程序化运动 |
| **Spore** | 身体编辑 + 部件决定能力 | 生态和 AI 深度不足；战斗偏轻 |
| **Screeps** | 玩家写行为逻辑让单位自主行动 | 纯程序员游戏；无身体构筑 |
| **The Bibites** | 数字生物 + 神经网络 + 自然选择 + 生态 | 无人工设计；玩家只能观察不能干预 |
| **The Sapling** | 身体编辑 + 生态模拟 + 物种进化 | 偏进化模拟，非任务制构筑 |
| **Creatures (1996)** | 数字生命 + 神经网络 + 繁殖 + 玩家教导 | 无任务/战斗；偏宠物养成 |

## 核心差异化

> **玩家设计身体 + 玩家设计 AI 蓝图 + 生物自主行动 + 地形生态位强绑定 + 确定性回放可分析**

与 The Bibites 的关键区别：The Bibites 是纯观察自然选择，玩家不能干预设计。本项目玩家是**主动设计者**，不是观察者。

## 玩家身份

边境实验站管理员 — 用半生物半机械的造物探索危险生态区，回收资源，解锁身体部件和 AI 模块。

## 首个可玩切片

完整愿景很大，但第一版只验证：

- 单体生物
- 预制 AI Profile（TaskFirst / Hunter / Coward）
- 单房间任务地图
- 回收任务
- 最小战后报告（Brain View + Mission Report）

**M3 的主观验收**：让 3 个非开发者看完一次任务，能复述"这只生物刚才为什么这么做"。技术上的 MVP 跑通不代表玩法上的 MVP 跑通。

## 目标玩家

这是一个**非常硬核的细分赛道**。竞品（Screeps、The Bibites）受众数千到数万级。

核心玩家画像：喜欢系统型游戏、享受设计和调试过程、对"AI 自主行为"有好奇心的玩家。介于 Screeps（纯程序员）和 Spore（纯休闲）之间。
