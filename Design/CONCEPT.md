# CONCEPT — Creature Builder Roguelite

> 版本 3.0 | 2026-05-28

## 一句话定位

设计生物的身体和AI，带它们深入危险生态区，活着带回尽可能多的东西。

> **设计身体，编排行为，深入生态，死或撤退，解锁重来。**

## 核心问题

**这次能走多远？**

不是"我收集了什么"，是"这次Run我能深入到第几区"。

## 核心循环

```
实验站：组装生物（身体部件 + AI蓝图）
↓
出发 → 第1个生态区（简单）
↓
收集资源/部件，遇到敌人
↓
决策：继续深入还是撤退？
↓
继续 → 更难的生态区
↓
...直到撤退或全灭
↓
Run结束：带回收集的资源/部件
↓
meta进度：解锁新部件/新AI规则/新生态区
↓
下次Run带更好的设计出发
```

## 关键设计

| 要素 | 设计 |
|------|------|
| **Run内资源** | 部件/信号/食物在生态区内获得，死了就没了 |
| **Meta资源** | 样本/数据死了也保留，用于解锁 |
| **撤退** | 随时可以撤退，带回当前所有收集。但放弃后面的奖励 |
| **全灭** | 丢失所有Run内资源。只保留样本和数据 |
| **解锁** | 增加下次Run的选项。不直接变强 |

## 设计支柱

### A. 身体决定能力

生物不是一张皮，而是一组功能部件。

```
腿决定移动方式
眼睛决定感知距离
爪子决定近战能力
甲壳决定防御
脑核决定AI复杂度
```

### B. AI决定性格

同一个身体，用不同AI蓝图会有完全不同的表现。

```
胆小型：远距离观察，残血逃跑
狂暴型：受伤后攻击欲上升，几乎不撤退
拾荒型：避战，优先吃尸体和捡资源
```

### C. 世界负责测试设计

每个环境都是一张"考试卷"。

```
酸性沼泽：测试抗性、路径选择
黑暗洞穴：测试听觉、嗅觉
机械废墟：测试潜行、电子干扰
```

### D. 失败必须有价值

死亡不只是惩罚，而是数据。

```
死亡原因 → 建议修改
回收资源 → 行为数据
```

### E. 涌现来自规则自由交互

世界有自己的生命。敌人/野生生物有自己的目标，环境独立运作。

### F. 撤退是策略

继续深入 = 高风险高回报
撤退 = 安全但放弃机会
全灭 = 丢失一切Run内资源

**每次撤退决策都是赌博。**

## 维度：等距 2.5D

**MVP起步参数**：相机角度55-60°，正交投影，单位高度1m。

> 参照系：**Hades**的视角 + **FTL**的Run结构 + **Spelunky**的程序化房间

## 硬核要素

| 要素 | 设计 |
|------|------|
| 死亡 | Run内生物死了就没了。Run内部件丢失 |
| 资源稀缺 | 信号是Run内资源，用完就没了 |
| 信息不完整 | 感知只给方向+强度，不给答案 |
| 选择有后果 | 继续还是撤退？赌不赌？ |
| 时间压力 | 环境在变化，窗口在关 |
