# CONCEPT — Creature Roguelike

> 版本 0.7 | 2026-05-27

## 一句话定位

一款 Auto Battler + Roguelite 游戏。你用探索获得的零件组装一支有性格的小队，让它们在危险环境中自动执行任务，你专注于编排和关键时刻的干预。

> **用手里的牌，打出最好的局。**

## 核心公式

```
稀缺零件 + 环境压力 → 倒逼策略编排 → 自动执行验证 → 成就感
```

## 核心循环

```
Run 开始：选择起始特征（决定本局方向）
↓
查看星图：选择下一个节点（精英/事件/商店/普通）
↓
进入节点：探索中获得随机零件
↓
用当前手牌组装/强化小队（策略决策）
↓
投放小队执行任务（自动执行，玩家观察）
↓
关键时刻用信号/扫描/路径标记干预（稀缺操作资源）
↓
成功 → 保留零件，小队成长
失败 → 失去部分零件，核心零件保留
↓
选择下一个节点，继续深入
```

## 这是什么游戏

**不是 Slay the Spire。** StS 是回合制卡牌，决策密度极高。

更接近：

| 参考 | 相似点 |
|------|--------|
| **Super Auto Pets** | 自动战斗 + 小队组装 + 零件协同 + 实时观察 |
| **Mechabellum** | 自动执行 + 编排策略 + 大单位对抗 |
| **Backpack Hero** | 物品管理 + 空间约束 + 零件组合 |
| **FTL** | Roguelike 分支星图 + 有限资源 + 实时干预 |
| **Hades** | Run 结构 + 永久成长 + 每局不同构筑 |

**核心体验接近 Super Auto Pets：组装小队 → 看它们打 → 结果由编排质量决定。**

## 四大设计支柱

| 支柱 | 一句话 |
|------|--------|
| **零件稀缺** | 你不能选你想要的，你只能用手里的牌打出最好的局 |
| **小队协同** | 深度不来自单体，来自多个性格之间的配合和冲突 |
| **压力倒逼策略** | 时间、敌人、背包容量，逼你在不完美条件下快速判断 |
| **失败驱动学习** | 核心零件不丢，但你会学到"这个编排在这种环境下不行" |

## 为什么 AI 自动执行不是缺点

不是"看 AI 表演"。而是：

- **我没法同时操控 3 个角色** — 操作占满注意力，没空想策略
- **AI 操控让我专注编排** — 谁前谁后、谁打谁捡、信号给谁
- **我的注意力全部放在**：队形、分工、信号时机

这就是 Auto Battler。编排的质量决定胜负，不是操作的速度。

## 核心风险

| 风险 | 解决 |
|------|------|
| 间接控制不如手操 | 手操做不了 — 3 个角色同时行动，策略才是核心 |
| 决策密度太低 | 小队制 + 多操作维度 + 分支星图 |
| 看 AI 走路无聊 | 任务短（20-40秒），性格冲突产生张力，战斗有可观察性 |
| 零件太随机挫败 | 核心零件死亡不丢，Run 开始有方向性选择 |
| 性格可预测就无聊 | AI 模块可矛盾叠加，组合产生第三种行为 |

## 玩家身份

拾荒者 / 探险者 — 在废弃实验设施和危险生态区中探索，收集零件，组装造物小队，完成任务，深入更危险区域。
