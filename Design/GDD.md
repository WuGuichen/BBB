# 1. 玩家身份

拾荒者 / 陪伴者。不是指挥官，是同伴。你和你创造的生命一起探险未知世界。

---

# 2. 核心循环

```
获得零件，组装生物（定义能力和性格）
↓
释放进入未知环境（战争迷雾，传感器是唯一信息来源）
↓
生物自主行动（探索/犹豫/战斗/逃跑/拾取）
↓
你通过它的感知"看到"世界
↓
关键节点喊话干预
↓
结果 → 理解它 → 强化它 → 进入更深的未知
```

## 决策密度

不是用 clicks/min 衡量。是用 **information updates / 10s** 衡量。

每 5-10 秒玩家必须从生物那里得到一条新信息（传感器揭示、停下、犹豫、发现、遭遇）。详细规范见 [Sensory_Design §7](Sensory_Design.md#7-信息密度要求)。

## 2.1 玩家核心动词（按时间占比）

| 动词 | 时间占比 | 操作 |
|------|---------|------|
| **观察** | 60-70% | 读生物状态、读环境、读节拍 |
| **指令/喊话** | 15-20% | 信号道具、建议、警告 |
| **组装** | 10-15% | 投放前的构筑决策 |
| **大决策** | 5% | 节点选择、零件取舍 |

**60-70% 时间在观察**——所以反馈系统的可读性是这游戏的生死。详见 [Sensory_Design §4](Sensory_Design.md#4-ai-反馈四层)。

## 2.2 嵌套循环：四层

**L1：5-10 秒微循环**
```
观察生物状态 → 预测下一步 → 它做了选择 → 验证或意外 → 学到新东西
```

每 5-10 秒一次。**这是密度的底线。** 如果中间 15 秒没有事件，地图设计不及格。

**L2：60-90 秒单次投放循环**
```
投放 → 5个核心瞬间 → 完成/失败 → 一句话总结
```

5 个核心瞬间详见 [Sensory_Design §6](Sensory_Design.md#6-五个核心瞬间)。

**L3：10-15 分钟单次 Run 循环**
```
进入区域 → 探索 → 组装调整 → 投放 → 结果 → 下一区域
× 3-5 次 → Run 结束
```

**L4：多次 Run 的认知循环**
```
假设（"配 X+Y 应该能解决 Z 情境"）→ 验证 → 更新世界模型
```

**L4 才是这游戏 30 小时玩下去的真正动力。** 没有 L4 这游戏就是 5 小时游戏。L4 要求 Mission Report 必须好——它是 L4 的输入端（见 §10.4）。

# 3. 感知与未知

详见 [Sensory_Design.md](Sensory_Design.md)。

核心要点：
- 三层信息：真相 / 生物认知 / 玩家认知
- 视觉 = 实景揭开（MVP）
- 嗅觉 = 气流方向，无精确位置（MVP）
- 听觉 = 声波涟漪，无方位精度（Post-MVP）
- 每种感知产生不同的 AI 反应
- 地图 = 信息节拍编排

# 4. 生物性格系统

## 4.1 性格由零件决定 `MVP`

装什么零件就产生什么性格。不需要单独设定。

## 4.2 AI 模块可矛盾叠加 `MVP`

```
只装勇气核心：看到敌人 → 冲
勇气 + 谨慎：高血量冲，低血量逃 → 中间血量犹豫
贪欲 + 谨慎：想拿资源但害怕 → 纠结
```

**矛盾叠加产生的犹豫是张力核心。**

## 4.3 性格的三类表现规格 `MVP`

每个 AI 模块必须定义**微行为 + UI 语言 + 音效倾向**三件套。设计师看得懂数值不算数，玩家看得懂才算数。

| 性格 | 微行为（无 UI 也能读） | UI 语言（高亮/图标） | 音效倾向 |
|------|------------------|---------------|---------|
| **勇气核心** | 朝威胁转身快、停顿短、冲刺加速明显 | 红色冲动箭头，怒气符号 | 短促向前、低吼 |
| **谨慎核心** | 停下扫描周围、绕边走、转向慢 | 蓝/灰警戒线，盾形图标 | 低频、迟疑、轻吸气 |
| **贪欲核心** | 路径向资源弯曲、绕开战斗去捡 | 金色吸引线，金币符号 | 轻快急促、咂嘴 |

**验证标准**：关掉所有 UI 和音效，**只看物理动作**，玩家能否说出"它是哪种性格"？如果不能，微行为不合格。

## 4.4 短期记忆 `MVP`

生物记住最近的遭遇：

```
被攻击过的方向 → 下次更警惕（朝向锁定时间 +50%）
发现过资源的位置 → 下次优先去
被你救过的经历 → 下次更听你的信号（信任度，见 §4.5）
```

**没有历史敏感，"理解我的生物"就是假的。**

## 4.5 信任度

`Trust v0 表现层：M3` `Trust v1 系统层：M6（Post-MVP）`

你和生物之间的关系是双向的。**v0.9 修正：表现层先于系统层**。

### Trust v0（表现层）`MVP M3`

一个内部 0.0-1.0 变量，只驱动**三处变化**（详见 [Sensory_Design §4.4](Sensory_Design.md#44-结果层-mvp--看懂关系记忆)）：
1. 犹豫时刻头顶图标变体（高信任 = 温和 `?`，低信任 = 不耐烦 `?!`）
2. 信号反馈短语的选择（"它点了点头" / "它迟疑地看了你一眼"）
3. Run 结束总结里一句变体

**v0 不影响 AI Goal priority**，只影响表现。

### Trust v1（系统层）`Post-MVP M6`

信任度直接进入 AI 决策：

```
你发信号救了它 → 信任度 +0.2 → 下次犹豫时更听你的（Goal priority shift 倍率提升）
你发信号坑了它 → 信任度 -0.3 → 下次不太听了
信任度低到一定程度 → 它完全忽略你的信号
```

**这是自走棋永远做不出来的深度 — 棋子没有记忆没有关系。**

# 5. 投放系统

## 5.1 MVP 锁定单生物 `MVP`

> **v0.9 决策**：MVP 阶段就一只生物。多生物不再是"M6 验证后回退"的方案。

理由：
- "陪伴一只生物"和"指挥多生物"是两个不同的核心动词，不能在同一个原型里同时验证
- 单生物天然支持命名、外观损伤、记忆、归来这些"它是我的生物"的设计（见 §13）
- 多生物的注意力分配玩法（Pikmin 路线）需要另一套 UI/UX/暂停机制

## 5.2 多生物 `Post-MVP`

留待陪伴感成立之后再开。届时讨论：
- 是否分裂成另一个产品/模式
- 还是作为本作的"远征模式"
- 半暂停机制是必要前提

# 6. 干预系统

## 6.1 干预是"喊话"，不是"操控" `MVP`

你是同伴，不是指挥官。喊话动词有人际感：

| 动词 | 形式 | 效果 | 限制 | 性格敏感 | 阶段 |
|------|------|------|------|----------|------|
| **警告** | 圈选区域 | AvoidThreat +30 | 信号道具 | 勇气核心可能忽略 | `MVP M4` |
| **建议（去那边）** | 点地图位置 | MoveToTarget +30 | 信号道具 | 谨慎核心可能不听 | `MVP M4` |
| **安抚** | 无目标 | Caution +20, Aggression -20 | 信号道具 | 勇气核心听不进 | `MVP M4` |
| **鼓舞** | 无目标 | Aggression +20, Caution -20 | 信号道具 | 谨慎核心听不进 | `MVP M4` |
| **召回** | 无目标 | 倾向返回出口 | 信号道具 | 勇气核心抵抗（部分执行） | `MVP M4` |
| **询问** | 无目标 | 生物分享当前感知（小幅扩散感知范围 1 次） | 短 CD | 一致响应 | `Post-MVP` |
| **指认** | 高亮某目标 | 生物注意它 | 短 CD | 性格敏感 | `Post-MVP` |

**询问是 MVP 最容易被忽视但最关键的动词**——它把玩家从"指挥官"变成"伙伴"。"我让我的生物告诉我它看到了什么"——这一句话就是这个游戏。

**每个动词在不同性格身上响应不同，且每次反馈进入 [Sensory_Design §4.4 结果层](Sensory_Design.md#44-结果层-mvp--看懂关系记忆)。** 这才有"沟通的不完美"感。

## 6.2 干预资源：两段经济 `MVP`

**关键原则：稀缺性和沟通感不能在同一个原型里同时验证。**

| 阶段 | 干预供给 | 测什么 |
|------|---------|--------|
| **M1-M4 Sandbox** | 每次投放固定 2-3 次喊话，无道具栏 | 沟通感（玩家是否觉得"我在和它对话"） |
| **M5+ Sandbox** | 加入战斗后仍保持固定喊话 | 干预在战斗节奏中是否成立 |
| **M7+ Roguelite** | 喊话变为道具：探索中获得，一次性使用，一局 0-3 个 | 稀缺性下的策略取舍 |

**M1-M4 不要测稀缺性**。先确保沟通感成立。

# 7. 零件系统 `MVP: M7`

## 7.1 零件是获得的，不是选择的

探索中随机获得。背包 8 格。满时必须丢弃。

## 7.2 零件类型 `MVP`

| 类型 | 示例 |
|------|------|
| 身体部件 | 快腿、厚甲、毒牙、嗅觉腔 |
| AI 模块 | 勇气核心、谨慎核心、贪欲核心 |
| 信号道具 | 警告标记、侦察探针、鼓励脉冲（在 Roguelite 经济下生效） |

## 7.3 核心零件 `MVP`

Run 开始选的起始特征是核心零件。**死亡不丢**。

> 这是 Hades 式的"灵魂保留"——你的生物有连续性。死亡是事件，不是清零。

## 7.4 Build Identity `MVP`

生物在 Run 内**持续存在**（不是每次投放重组）。零件装上就留下。玩家能说出"我这只生物是侦察型"。

# 8. 地图与环境

## 8.1 小而密的迷雾房间 `MVP`

不是大而空的地图。是小而密、每 5-10 秒有信息事件的房间。详见 [Sensory_Design §8](Sensory_Design.md#8-地图--信息节拍编排)。

**单房间目标尺寸**：20×20m，包含 5-10 个信息事件。

## 8.2 环境变量 `MVP`

| 变量 | 影响 |
|------|------|
| 光照 | 视觉范围变化（亮区大/暗区小） |
| 噪音 | 听觉检测范围变化（Post-MVP） |
| 气味浓度 | 嗅觉检测范围变化 |

## 8.3 战争迷雾 `MVP`

生物走到哪里揭开哪里。已揭开区域的"记忆衰减"规则见 [Sensory_Design §1.1](Sensory_Design.md#11-已揭开区域的记忆衰减)。

## 8.4 视角与相机

详见 [TECHNICAL §11](TECHNICAL.md)。**2.5D 等距俯视**，固定 55-60° 相机角度。

# 9. 战斗

## 9.1 战斗是"事件"，不是"高潮" `MVP`

**反直觉的设计**：战斗短而突然，3-5 秒接触，结果立刻可解释。

```
生物遇到敌人
→ 0.3 秒前摇（情绪层 `!`）
→ 3 秒接触（扣血 + 音效 + 伤害数字）
→ 一方死亡或撤退
→ 继续探索
```

战斗的紧张感不来自打击感，来自：
- "它会撑住吗？"
- "它会撤吗？"
- "我要不要喊它撤？"

**这些都是性格读取，不是动作读取。**

## 9.2 战斗中的反馈

战斗也走 [Sensory_Design §4 四层反馈](Sensory_Design.md#4-ai-反馈四层)：

- 行为层：朝向锁定 + 加速接触
- 情绪层：`!` 出现
- 理由层：底部日志"它选择了进攻（勇气核心）"
- 结果层：胜/负 + 是否听了你的撤退信号

# 10. 反馈与 UI

**原则：信息全部 in-world，菜单越少越好。**

## 10.0 HUD 布局

```
┌─────────────────────────────────────────────┐
│  [Run 进度]               [当前区域目标]      │  ← 顶部极简（5% 信息）
│                                              │
│       ┌──┐    ← 生物头顶状态                  │
│       │❓│    （朝向、状态图标、感知图标）    │
│       └──┘                                   │
│                                              │
│         🟢                                   │
│        (生物)                                │
│                                              │
│  "它选择了进攻（勇气核心）"  ← 底部行为日志    │
│                                              │
│  [警告] [建议] [召回] [询问]  [慢放] [切换]   │  ← 底部工具栏
└─────────────────────────────────────────────┘
```

**生物头顶状态 = 主要信息源**（70% 信息）
- 朝向箭头（朝哪走）
- 当前 State 图标（探索/警戒/犹豫/逃跑/拾取/战斗）—— 6 个图标
- 当前活跃感知（眼睛/鼻子/耳朵 + 强度条）
- HP 条（半透明，不抢戏；HP 低时变红+轻微闪烁）

**屏幕底部行为日志**（25% 信息）
- 性格主导的关键决策，一句话
- 每 5-10 秒一条，不刷屏
- 这是"理解"的实时反馈

**Run 进度条**（5% 信息）—— 极简，不抢戏

## 10.1 感知可视化

详见 [Sensory_Design.md](Sensory_Design.md)。

## 10.2 AI 反馈四层 `MVP`

详见 [Sensory_Design §4](Sensory_Design.md#4-ai-反馈四层)。

| 层 | 例 |
|---|----|
| 行为层 | 减速、转身、扫描 |
| 情绪层 | `?` `!` `★` |
| 理由层 | "它选择了进攻（勇气核心）" |
| 结果层 | "它点了点头" |

## 10.3 犹豫时刻 `MVP`

`?` + 减速 + Brain View 浮现 + 慢放 0.7x + 反向高亮。详见 [Sensory_Design §5](Sensory_Design.md#5-犹豫的表现规范)。

## 10.4 Mission Report `MVP M6`

不是表格，是**生物视角的回放叙事**：

```
[回放 1] 0:08  我嗅到了气味，但你没让我去查
[回放 2] 0:23  我在分叉点犹豫了，因为...
[回放 3] 0:41  我冲向了敌人，因为你装了勇气核心
[回放 4] 0:58  我被偷袭，因为我没看到左侧的洞
[结论]    下次：装嗅觉腔，或选谨慎核心
```

**这是认知循环的输入端**——它把这次的体验变成下次的假设。

# 11. 关键风险

| 风险 | 解决 |
|------|------|
| 探索节奏太慢 | 地图小而密，信息事件每 5-10 秒（[Sensory §6](Sensory_Design.md#6-信息密度要求)） |
| 看不懂生物在想什么 | AI 四层反馈（[Sensory §4](Sensory_Design.md#4-ai-反馈四层)） |
| 战斗抢走注意力 | 短而突然（3-5 秒），定位是事件 |
| 陪伴幻想立不起来 | Trust v0 表现层 M3 就要可见 |
| 性格可预测就无聊 | 矛盾叠加 + 短期记忆 + 环境敏感 |
| 性格只有数值，没有视觉表现 | §4.3 三表现规格强制定义 |

# 13. 生物作为角色 `MVP`

> v0.9 新增。**玩家爱上的不是系统，是那只东西**。下面这些不昂贵，但能把"AI 单位"变成"我的生物"。

| 项 | 说明 | 阶段 |
|---|------|------|
| **命名** | Run 开始时给生物起名（或抽随机名） | `MVP M2` |
| **伤痕** | 受过重伤后占位体留色块/缺口，跨投放保留 | `MVP M5` |
| **归来反应** | 任务完成回到出口时，1 秒短暂"靠近玩家"动作（朝镜头微转） | `MVP M2` |
| **记忆方向** | 死过同伴的位置/被攻击过的方向：进入这片区域时朝向锁定 | `MVP M4`（短期记忆触发） |
| **死亡纪念** | 死亡时屏幕中央停留 2 秒："{名字} 在第 N 个房间倒下了" | `MVP M5` |
| **Run 结束总结** | 一句性格总结："它三次闻到威胁，两次选择绕开；最后一次听了你的警告" | `MVP M7` |
| **它在睡觉** | 投放间隙生物有 idle 动画（伏地呼吸） | `Post-MVP` |
| **它认得你** | 高信任时投放瞬间朝镜头偏头 | `Post-MVP`（依赖 Trust v1） |

**验收原则**：M2 Playtest 必问 **"你愿不愿意再派同一只生物出去？"** 如果玩家说"换一只也行"，本章节内容不够，回看上表。
