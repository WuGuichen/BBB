# 1. 玩家身份

**荒野生物工程师** — 边境实验站的管理员。

玩家在废土上建立实验站，通过回收基因、骨骼、机械部件，创造适应环境的生物去探索危险生态区。

**玩家不在世界里。** 你只能通过你创造的生物去"看"这个世界。

---

# 2. 核心循环

```
1. 接受任务 / 选择生态区
2. 分析环境威胁
3. 设计或选择生物（身体部件 + AI蓝图）
4. 投放生物
5. 间接干预 / 观察行动
6. 回收资源和数据
7. 分析失败或成功原因
8. 解锁新部件、新AI规则、新环境
9. 继续改造
```

## 2.1 玩家核心动词（按时间占比）

| 动词 | 时间占比 | 操作 |
|------|---------|------|
| **设计** | 30-40% | 选部件、配AI蓝图、调参数 |
| **观察** | 30-40% | 读生物状态、读环境、读反馈 |
| **干预** | 10-15% | 信号工具、环境工具 |
| **分析** | 10-15% | 战后报告、改造方向 |

**设计和观察是核心。** 不是手操，是设计系统让它自己跑。

## 2.2 嵌套循环

**L1：30秒微循环**
```
观察生物状态 → 预测下一步 → 它做了选择 → 验证或意外 → 学到新东西
```

**L2：5-12分钟单次任务循环**
```
投放 → 生物自主行动 → 意外发生 → 干预/观察 → 结果 → 战后报告
```

**L3：30分钟成长循环**
```
完成任务 → 解锁部件/AI规则 → 改造生物 → 测试新设计 → 打开新生态区
```

**L4：多次Run的认知循环**
```
假设（"这个部件组合应该能解决这个环境"）→ 验证 → 更新设计
```

# 3. 身体部件系统

## 3.1 部件类型

生物由 8 类部件组成：

### Core 核心

决定基础形态和能量容量。

| 类型 | 特点 |
|------|------|
| 轻型核心 | 便宜、轻、脆 |
| 重型核心 | 血厚、慢、耗能高 |
| 群体核心 | 适合多单位协作 |
| 捕食核心 | 攻击欲和追踪能力强 |
| 实验核心 | 可装更多AI模块，但不稳定 |

### Locomotion 移动器官

| 类型 | 特点 |
|------|------|
| 双足 | 均衡，适合操作工具 |
| 四足 | 稳定，适合追猎 |
| 六足 | 地形适应强，速度中等 |
| 蛇形 | 可钻缝，转向慢 |
| 翅膀 | 跨越地形，能耗高 |

### Sensor 感知器官

| 类型 | 特点 |
|------|------|
| 普通眼 | 基础视觉 |
| 夜视眼 | 黑暗可见 |
| 热感器 | 发现隐藏生物 |
| 大耳 | 听觉范围大 |
| 嗅觉腔 | 追踪血迹/尸体/巢穴 |
| 危险腺体 | 感知毒气、火、酸 |
| 群体天线 | 共享同伴情报 |

### Weapon 攻击器官

爪、牙、角、毒刺、喷酸腺、电击器、远程骨刺、机械锯

### Defense 防御器官

甲壳、鳞片、脂肪层、再生组织、反伤尖刺、热抗皮肤、酸抗脚垫、伪装皮肤

### Utility 功能器官

携带囊、挖掘爪、采集口器、开门工具、能量电池、隐身膜、跳跃囊、攀爬钩

### Brain 脑模块

控制AI复杂度。

| 类型 | 特点 |
|------|------|
| 原始脑 | 只能执行少量规则 |
| 猎手脑 | 攻击、追踪、伏击加成 |
| 群体脑 | 队友协作规则更多 |
| 工蜂脑 | 采集、搬运、建造加成 |
| 战术脑 | 可使用掩体、包抄、撤退 |

### Social 社交/群体器官

信息素腺体、吼叫器官、信号天线、群体同步神经、恐吓面具

## 3.2 部件属性

每个部件带标签：

```
重量、能耗、噪音、耐久、维修难度
生物质消耗、金属消耗
恐惧影响、攻击倾向影响
适配地形、兼容部件、AI需求
```

## 3.3 预算系统

防止堆满最强部件：

```
体重预算、能量预算、脑容量预算
突变稳定度、部件槽位、维护成本
```

重甲+大脑+六足+远程武器 = 速度慢、饥饿快、维护贵。

**构筑有取舍。**

# 4. AI蓝图系统

## 4.1 玩家看到的不是代码，而是规则卡

```
当【看到敌人】
并且【敌人距离 < 5m】
并且【自己血量 > 50%】
则【近战攻击倾向 +60】
```

玩家组合：条件卡、比较卡、意图卡、权重卡、冷却卡、状态卡。

## 4.2 AI蓝图结构

一个AI Blueprint包含：

```
感知权限
黑板变量
状态机
Utility规则
行为意图
记忆规则
群体通信规则
```

## 4.3 基础状态

```
Idle       空闲
Explore    探索
Forage     觅食
Combat     战斗
Flee       逃跑
Recover    恢复
Return     返回
Protect    护卫
Nest       筑巢/驻守
```

玩家可以调状态切换条件。

## 4.4 意图 Intent

```
MoveToTarget
AttackTarget
FleeFromThreat
EatFood
InvestigateStimulus
ReturnToNest
ProtectAlly
CarryItem
UseAbility
Hide
CallForHelp
Rest
```

玩家不直接调用动作，只提高或降低某个Intent的分数。

## 4.5 Utility打分示例

### "怂但聪明的拾荒者"

```
FleeScore = ThreatNearby*60 + SelfHealthLow*80 + Alone*30 - AllyNearby*25
ScavengeScore = CorpseNearby*70 + Hunger*40 - ThreatNearby*50
ReturnScore = InventoryFull*80 + SelfHealthLow*40 + NightTime*20
```

表现：平时找尸体，看到强敌就跑，装满就回家。

### "狂暴猎手"

```
AttackScore = EnemyVisible*70 + EnemyWounded*40 + SelfAnger*50 - SelfHealthLow*20
FleeScore = SelfHealthCritical*40 - SelfAnger*50
```

表现：容易追击，受伤后更凶，很少撤退。

## 4.6 内部需求

行为 = f(AI蓝图, 需求, 环境, 记忆)，不只是f(AI蓝图)。

| 需求 | 范围 | 行为影响 |
|------|------|---------|
| 饥饿 | 0-100 | >70时优先找食物，可能忽略信号 |
| 疲劳 | 0-100 | >80时减速，可能找地方休息 |
| 恐惧 | 0-100 | >60时逃跑优先级上升 |
| 好奇 | 0-100 | >50时会被未知吸引 |
| 攻击欲 | 0-100 | 受伤累积，HP越低越高 |
| 思乡 | 0-100 | 远离出口太久，>70时倾向返回 |

详细规范见 [World_Simulation §3](World_Simulation.md#3-内部需求系统)。

# 5. 玩家介入工具

## 5.1 信号工具

| 信号 | 效果 | 稀缺度 |
|------|------|--------|
| 移动信标 | 提高前往区域倾向 | 有限 |
| 危险标记 | 提高避开区域倾向 | 有限 |
| 猎物标记 | 提高攻击某目标倾向 | 有限 |
| 召回信标 | 提高返回倾向 | 有限 |
| 安抚波 | 降低恐惧/愤怒 | 稀有 |
| 刺激波 | 提高攻击/速度，增加失控风险 | 稀有 |

**信号不直接控制行为，而是改变Utility分数。** AI设计太极端，生物可能不听。

## 5.2 环境工具

投放诱饵、投放食物、制造噪音、释放气味、点火、关灯、打开门、放置临时巢穴

## 5.3 紧急工具

紧急传送（救回生物但部件损坏）、镇静剂（让失控生物停止）、自毁命令（牺牲生物保留数据）

# 6. 任务系统

## 6.1 任务类型

| 类型 | 目标 | 测试什么 |
|------|------|---------|
| **回收任务** | 进入危险区，找到资源，带回 | 寻路、生存、撤退逻辑 |
| **生态观察** | 观察野生生物，记录行为数据 | 潜行、感知、距离控制 |
| **狩猎任务** | 击杀或捕获目标 | 攻击策略、追踪、损耗控制 |
| **护送任务** | 保护脆弱单位穿过危险区 | 队伍AI、保护规则、队形 |
| **污染清理** | 清除污染源，修复生态节点 | 环境抗性、长时间续航 |
| **适应性试炼** | 在特定环境存活/不杀生完成采集 | 理解系统 |

## 6.2 30秒微循环

```
我的生物发现了什么？
它为什么这么做？
它现在安全吗？
我要不要发出信号？
它的AI是否误判？
是否该撤离？
```

# 8. 战后报告

任务结束必须有完整报告：

```
任务结果：失败
目标：回收古代电池
失败点：生物接近电池前被酸蚀虫包围

关键行为：
00:41 听到声音 → 调查
01:03 发现尸体 → 进食
01:19 被酸蚀虫攻击 → 战斗
01:24 血量低于35% → 逃跑
01:28 体力耗尽 → 死亡

问题：
- 进食优先级过高，偏离任务路径
- 没有"任务物品优先"规则
- 缺少酸抗性脚部

建议：
- 添加规则：任务目标可见时，目标优先级+80
- 添加规则：体力<30%时避免危险地形
- 替换脚部为"酸蚀足垫"

回收资源：破损腿部x2，酸蚀甲壳x1，行为数据x36
```

**这是玩家真正成长的地方。** 看完报告会想"我知道怎么改了"。

# 9. 感知与未知

详见 [Sensory_Design.md](Sensory_Design.md)。

核心要点：
- 三层信息：真相 / 生物认知 / 玩家认知
- 视觉 = 实景揭开
- 嗅觉 = 气流方向，无精确位置
- 听觉 = 声波涟漪，无方位精度（Post-MVP）
- 每种感知产生不同的AI反应
- AI反馈四层：行为/情绪/理由（推断）/结果

# 12. 战斗

## 战斗是"事件"，不是"高潮"

```
生物遇到敌人
→ 0.3秒前摇（情绪层 !）
→ 3秒接触（扣血 + 音效 + 伤害数字）
→ 一方死亡或撤退
→ 继续探索
```

战斗的紧张感来自：
- "它会撑住吗？"
- "它会撤吗？"
- "我要不要喊它撤？"

**这些都是AI读取，不是动作读取。**

# 13. 玩家构筑示例

## "夜行拾荒者"

```
Core：轻型核心
Movement：四足
Sensor：夜视眼 + 嗅觉腔
Weapon：小型毒牙
Defense：伪装皮肤
Utility：携带囊
Brain：拾荒脑
AI：避战拾荒蓝图
Traits：胆小、夜行、低噪音
```

表现：夜间行动，找尸体和资源，看到敌人就跑。

## "狂暴猎手"

```
Core：捕食核心
Movement：四足
Sensor：热感器 + 大耳
Weapon：爪 + 毒刺
Defense：鳞片
Brain：猎手脑
AI：狂暴攻击蓝图
Traits：受伤后攻击欲上升
```

表现：追击猎物，受伤后更凶，容易死但爆发强。

# 14. 关键风险

| 风险 | 解决 |
|------|------|
| 玩家看不懂AI在干什么 | 强反馈：意图UI + 决策解释 + 战后日志 |
| 不能直接控制很烦 | 有限但强力的间接指挥，不听话有合理原因 |
| 构筑太复杂新手进不来 | 预设模板 + 卡片规则 + 失败后建议 |
| 所有玩家找到最优解 | 环境多样化 + 生态反制 + 资源约束 |
| 涌现失败时什么都没发生 | 活物密度达标（3-5个独立AI主体） |
| 调试成本爆炸 | Decision Trace + Seed重放 |
