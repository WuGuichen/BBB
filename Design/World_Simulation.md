# WORLD SIMULATION — 多智能体仿真规范

> 版本 1.0 | 2026-05-28
>
> 这是游戏涌现性的核心文档。设计师定义规则，不定义剧本。系统自己跑，结果自己出。

---

# 1. 设计原则

## 1.1 规则优先，剧本归零

| 做法 | 例子 |
|------|------|
| ✅ 定义规则 | 酸池每秒扩散 0.5m |
| ✅ 定义实体 | 房间里有 3 只小生物 + 1 只巡逻者 + 2 具尸体 |
| ✅ 定义倾向 | 饥饿 > 70 时优先找食物 |
| ❌ 定义剧本 | 0:08 嗅觉触发，0:15 生物转向左边 |
| ❌ 定义节拍序列 | 每 5-10 秒一个事件 |

**设计师的工作**：放活物、调参数、观察生态。
**不是**：安排每一秒发生什么。

## 1.2 涌现来源

涌现 = 多个独立规则自由交互 → 设计师未预期的结果。

本游戏的涌现来自 5 层：

| 层 | 来源 | 你能看到什么 |
|---|------|------------|
| **① 多智能体目标冲突** | 敌人/野生生物有自己的目标，不只为玩家服务 | 敌人在打别的东西，你悄悄通过 |
| **② 环境状态连锁** | 酸池扩散、毒气蔓延、火源点燃 | 酸池逼走敌人 → 敌人撞墙 → 墙塌堵路 |
| **③ 生物内部需求** | 饥饿/疲劳/恐惧/好奇/攻击欲/思乡 | 生物饿了去吃东西，没听你的信号 |
| **④ 跨 Run 持久化** | 死过的位置、杀过的敌人、走过的路 | 第 5 次 Run，地图已被你的历史改变 |
| **⑤ 信息推断** | 默认只显示行为，因果靠玩家推断 | "它为什么去那边？"——自己想 |

# 2. 多智能体仿真

## 2.1 每个实体独立运作

所有活物（敌人、野生生物、清道夫、玩家生物）共用同一套 AI Planner，各自有独立的：

```csharp
public sealed class AgentProfile
{
    public string AgentId;
    public AgentType Type;           // PlayerCreature / Enemy / Wildlife / Scavenger
    public AiModuleDefinition[] Modules;
    public NeedState Needs;          // 饥饿/疲劳/恐惧/好奇/攻击欲/思乡
    public MemoryState Memory;       // 短期记忆
    public Goal CurrentGoal;         // 当前目标（由 Planner 计算）
    public AgentSchedule Schedule;   // 活动节律（日行/夜行/随机）
}
```

**关键设计**：
- 所有实体共用 Planner，不硬编码行为
- 每个实体只管自己的目标，不管其他实体
- 交互是"碰巧发生的"，不是设计师安排的

## 2.2 敌人有自己的目标

| 敌人类型 | 自己的目标 | 与玩家的关系 |
|---------|-----------|------------|
| **巡逻者** | 守卫领地，驱逐入侵者 | 玩家是入侵者之一，不是唯一目标 |
| **捕食者** | 饿了找猎物（可能是小生物，不一定是玩家） | 玩家只是众多猎物候选 |
| **清道夫** | 找尸体/食物残渣 | 玩家杀的敌人会引来清道夫 |
| **群体** | 互相靠近，领地防御 | 一个被惊动 → 其他也警戒 |
| **嗅觉猎手** | 靠气味追踪 | 生物装嗅觉腔 → 气味扩散更广 → 可能被误导 |

**关键设计**：敌人 AI 在玩家不在场时也运行（低 tick 频率简化版）。

```csharp
// 敌人不只追玩家，有自己的目标优先级
public enum EnemyGoal
{
    Patrol,         // 巡逻
    Hunt,           // 捕猎（可能是玩家，可能是小生物）
    Eat,            // 进食（有猎物尸体时）
    Rest,           // 休息（受伤/疲劳时）
    Flee,           // 逃跑（HP 低 + 恐惧高）
    GuardTerritory, // 守卫领地（感知到入侵时）
}
```

## 2.3 野生生物

房间里除了敌人，还有野生小生物：

| 类型 | 行为 | 涌现作用 |
|------|------|---------|
| **小爬虫** | 觅食，受惊逃跑 | 被玩家生物吓到 → 逃跑方向可能暴露藏起来的敌人 |
| **清道夫** | 跟着尸体/食物走 | 你杀的敌人引来清道夫 → 清道夫可能挡住你的路 |
| **共生体** | 跟着大生物走 | 敌人身边有小跟班 → 打小的会惊动大的 |

**每个房间至少 2-3 个野生生物**，确保"有活物在动"。

# 3. 内部需求系统

## 3.1 需求定义

每个生物（包括玩家生物）有 6 个内部需求：

| 需求 | 范围 | 行为影响 | 玩家可见？ |
|------|------|---------|-----------|
| **饥饿** | 0-100 | 饥饿 > 70 时优先找食物，可能忽略玩家信号 | 头顶图标：空盘子 |
| **疲劳** | 0-100 | 疲劳 > 80 时减速，可能找地方休息 | 行为层：减速 |
| **恐惧** | 0-100（短期） | 恐惧 > 60 时逃跑优先级上升 | 情绪层：颤抖 |
| **好奇** | 0-100 | 好奇 > 50 时会被未知吸引 | 行为层：朝未知方向扫 |
| **攻击欲** | 0-100 | 受伤累积，HP 越低攻击欲越高 | 行为层：加速冲撞 |
| **思乡** | 0-100 | 远离出口太久累积，> 70 时倾向返回 | 行为层：回头看出口方向 |

## 3.2 需求 × 性格 = 意外行为

```
勇气核心 + 饥饿 80 + 左边有食物 + 右边有敌人
→ 去吃东西（饥饿压过勇气）
→ 玩家预期它冲敌人，但它去吃了
→ 这不是设计师安排的"意外"，是规则碰撞
```

**行为 = f(性格, 需求, 环境, 记忆)**

不是 f(性格) 的单一函数。

## 3.3 需求的视觉表现

| 需求 | 头顶 | 行为层 |
|------|------|--------|
| 饥饿高 | 空盘子图标 | 朝食物方向路径弯曲 |
| 疲劳高 | Zzz | 减速，可能停下 |
| 恐惧高 | ~~ | 身体压低，颤抖 |
| 好奇高 | ? 缓慢 | 朝未知方向轻扫 |
| 攻击欲高 | 怒气 | 加速冲撞 |
| 思乡高 | ← 出口箭头 | 回头看出口 |

**不显示具体数值**——玩家只能从行为推断状态。

## 3.4 需求的交互

需求之间可以互相影响：

```csharp
// 受伤 → 恐惧上升 + 攻击欲上升
// 看到食物 → 饥饿下降 + 好奇下降
// 远离出口 → 思乡上升 + 恐惧上升（如果已受伤）
// 和玩家互动 → 恐惧下降（如果信任高）/ 恐惧上升（如果信任低）
```

# 4. 环境规则

## 4.1 独立运作的环境实体

每个环境元素有自己的规则，不管谁在附近：

| 实体 | 自己的规则 | 不管什么 |
|------|-----------|---------|
| **酸池** | 每秒扩散 0.5m，遇墙停止 | 不管生物/敌人在哪 |
| **毒气** | 风向驱动，每 5 秒变向 | 不管谁在呼吸 |
| **火源** | 点燃可燃物，蔓延 | 不管蔓延方向有没有敌人 |
| **水流** | 推动实体移动 | 不管是生物还是敌人 |
| **可破坏墙** | 被撞 3 次倒塌 | 不管倒塌会砸到谁 |
| **食物源** | 被吃后 60 秒再生 | 不管谁在等 |

## 4.2 环境连锁示例

```
酸池扩散 → 巡逻者被迫移动 → 巡逻者撞到可破坏墙 → 墙塌堵路
→ 玩家生物被迫绕路 → 绕路时发现新资源
设计师没安排这个，是规则碰巧发生的。
```

```
毒气变向 → 气味被吹散 → 嗅觉检测范围变化 → 生物失去目标
→ 生物停下犹豫 → 玩家发信号 → 信号到达时毒气又吹回来
环境影响了干预的有效性。
```

## 4.3 气味环境

气味不是设计师放的"节拍元素"，是**活物和物体自然散发的**：

```csharp
// 每个实体根据类型散发气味
public sealed class SmellEmitter
{
    public SmellType Type;      // Threat / Food / Decay / Player
    public float Intensity;     // 基础强度
    public float Range;         // 基础范围
    public float DecayRate;     // 衰减速率
}

// 尸体 → Decay 气味 → 吸引清道夫
// 敌人 → Threat 气味 → 被嗅觉检测
// 食物 → Food 气味 → 被饥饿生物检测
// 玩家生物 → Player 气味 → 被嗅觉猎手追踪
```

**气味是环境状态的自然表达，不是设计师放的标记。**

# 5. 跨 Run 持久化

## 5.1 世界记忆

地图是"长期场所"。Run 之间发生的事会留下：

| 事件 | 持久化效果 | 消退 |
|------|-----------|------|
| 生物死亡 | 尸体保留 3-5 个 Run | 之后被清道夫吃完 |
| 敌人死亡 | 尸体保留 2-3 个 Run | 吸引清道夫，改变区域生态 |
| 资源未拿走 | 保留 | 几个 Run 后被清道夫吃掉/被其他生物拿走 |
| 触发陷阱 | 已触发状态保留 | 部分重置，部分不重置 |
| 走过路径 | 气味痕迹保留 1-2 个 Run | 被嗅觉猎手追踪 |

## 5.2 生态演变

跨 Run 后，地图生态会变化：

```
Run 1：你在 A 区杀了 2 只巡逻者
Run 2：A 区清道夫聚集（吃尸体），巡逻者减少
Run 3：尸体被吃完，清道夫离开，新的巡逻者可能迁入
Run 4：A 区恢复，但地形被你上次破坏的墙改变了
```

**玩家对世界有持久影响，不只是"重开一局"。**

## 5.3 存储

```csharp
public sealed class WorldState
{
    public Dictionary<string, EntitySnapshot> DeadEntities;     // 死亡实体位置和类型
    public Dictionary<string, EnvironmentDelta> EnvChanges;     // 环境变化（塌墙/酸池痕迹）
    public List<SmellTrace> SmellTraces;                        // 气味痕迹
    public int TotalRuns;                                       // 总 Run 数
    public Dictionary<string, int> KillHistory;                 // 击杀历史
}
```

# 6. 反馈层：从"告知"到"推断"

## 6.1 两层反馈

**默认层**：只显示行为和状态图标，不解释因果。
```
它冲向了敌人。
（头顶图标：低 HP / 听到威胁 / 看到敌人 / 饥饿低）
```

**追问层**（玩家按 R 键或点击生物）：显示因果归因。
```
你按了 R：
→ 勇气核心 +40
→ 嗅觉威胁 +20
→ 饥饿 +5
→ 主导：进攻
```

**想看的人能看到，不想看的人可以纯靠感觉玩。**

## 6.2 Mission Report 改造

从"生物视角的回放叙事"改为**数据轨迹**：

```
任务结束：
- 走了 87m
- 嗅过 4 次
- 停下 3 次
- 和 1 个野生生物擦肩
- 绕过 1 个威胁
- 拾取 1 个零件
- 未使用信号

你想："下次让它带毒牙看会不会主动打巡逻者。"
```

**不是"它说它为什么这样做"，是"你自己看数据想"。**

# 7. 活物密度规范

涌现的前提是"房间里有足够的活物在动"。

## 7.1 最小密度

| 房间类型 | 最小活物数 | 包含 |
|---------|-----------|------|
| 探索室 | 3-4 | 1-2 小生物 + 1 清道夫 + 环境元素 |
| 遭遇室 | 4-5 | 1-2 敌人 + 1-2 小生物 + 环境元素 |
| 资源室 | 3-4 | 1 敌人守卫 + 1-2 清道夫 + 环境元素 |
| 转场室 | 2-3 | 1 巡逻者 + 环境元素 |

**低于这个密度 = 涌现不够，房间太空。**

## 7.2 活物放置规则

设计师放活物，不管它们什么时候互动：

```csharp
public sealed class RoomSpawnConfig
{
    public List<AgentSpawn> Spawns;    // 放什么活物
    public List<EnvEntity> EnvEntities; // 放什么环境元素
    // 不定义：它们什么时候互动、在哪里互动
}
```

# 8. M1 验收标准（涌现版）

**不再是"跑通 60 秒剧本"，是"同一房间播放 3 次，事件序列必须显著不同"。**

## 8.1 可测量指标

- 3 次播放中，至少出现 5 种不同的"主要事件"（拾取/战斗/逃跑/嗅探/拐弯/犹豫/觅食/...）
- 没有任何 2 次播放有 80% 相同的事件序列
- 每次播放至少有 1 个"设计师未预料"的事件（Decision Trace 标记 = 高 Utility 但非剧本预期）

## 8.2 主观验收

- 3 个非开发者各看一次，每人说出的故事都不一样
- 任意 1 个玩家看 2 次后，能说"这两次完全不同"
- 任意 1 个玩家看 3 次后，能说"我下次想试试 X，看会不会发生 Y"
  - 这个 X 和 Y 不能是开发者预料过的

## 8.3 不通过的处理

| 反馈 | 原因 | 应对 |
|------|------|------|
| "三次都差不多" | 活物不够/规则太简单 | 加活物，加需求交互 |
| "什么都没发生" | 活物密度太低 | 每个房间至少 3-4 个活物 |
| "太乱了看不懂" | 多智能体互相干扰 | 降低活物密度，简化需求 |
| "我不关心发生了什么" | 陪伴感不够 | 回看 GDD §13 生物角色化 |
