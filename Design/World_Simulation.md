# WORLD SIMULATION — 多智能体仿真与生态规范

> 版本 2.0 | 2026-05-28
>
> 世界有自己的生命。设计师定义规则，不定义剧本。

---

# 1. 设计原则

## 1.1 规则优先，剧本归零

| 做法 | 例子 |
|------|------|
| ✅ 定义规则 | 酸池每秒扩散0.5m |
| ✅ 定义实体 | 房间里有3只小生物+1只巡逻者+2具尸体 |
| ✅ 定义倾向 | 饥饿>70时优先找食物 |
| ❌ 定义剧本 | 0:08嗅觉触发，0:15生物转向左边 |

## 1.2 涌现来源

| 层 | 来源 |
|---|------|
| **① 多智能体目标冲突** | 敌人/野生生物有自己的目标 |
| **② 环境状态连锁** | 酸池扩散、毒气蔓延、火源点燃 |
| **③ 生物内部需求** | 饥饿/疲劳/恐惧/好奇/攻击欲/思乡 |
| **④ 跨Run持久化** | 死过的位置、杀过的敌人、走过的路 |
| **⑤ 信息推断** | 默认只显示行为，因果靠玩家推断 |

# 2. 多智能体仿真

## 2.1 每个实体独立运作

所有活物共用同一套AI Planner，各自有独立的：

```csharp
public sealed class AgentProfile
{
    public string AgentId;
    public AgentType Type;           // PlayerCreature / Enemy / Wildlife / Scavenger
    public AiModuleDefinition[] Modules;
    public NeedState Needs;
    public MemoryState Memory;
    public Goal CurrentGoal;
}
```

## 2.2 生态位

每种野生生物有生态角色：

| 生态位 | 行为 | 涌现作用 |
|--------|------|---------|
| **草食者** | 吃植物，繁殖快 | 群体大时吸引捕食者 |
| **小型捕食者** | 猎杀弱者 | 可能追玩家生物，也可能追小虫 |
| **大型捕食者** | 领地强，数量少 | 守在关键位置 |
| **清道夫** | 吃尸体 | 你杀的敌人引来清道夫 |
| **寄生种** | 附着宿主，改变行为 | 被寄生的生物行为异常 |
| **筑巢者** | 改变地图结构 | 堵路/开路 |
| **机械巡逻者** | 按路线行动 | 掉落金属，可预测 |
| **异常体** | 规则特殊 | 强但不可预测 |

## 2.3 敌人有自己的目标

| 敌人类型 | 自己的目标 | 与玩家的关系 |
|---------|-----------|------------|
| 巡逻者 | 守卫领地 | 玩家是入侵者之一 |
| 捕食者 | 饿了找猎物 | 玩家只是众多猎物候选 |
| 清道夫 | 找尸体/食物 | 玩家杀的敌人引来它 |
| 群体 | 互相靠近，领地防御 | 一个被惊动→其他也警戒 |

**关键设计**：敌人AI在玩家不在场时也运行（低tick频率）。

## 2.4 野生生物

每个房间至少2-3个野生生物：

| 类型 | 行为 | 涌现作用 |
|------|------|---------|
| 小爬虫 | 觅食，受惊逃跑 | 被吓到可能暴露藏起来的敌人 |
| 清道夫 | 跟着尸体/食物走 | 你杀的敌人引来它 |
| 共生体 | 跟着大生物走 | 打小的会惊动大的 |

# 3. 内部需求系统

每个生物有6个内部需求：

| 需求 | 范围 | 行为影响 |
|------|------|---------|
| 饥饿 | 0-100 | >70时优先找食物 |
| 疲劳 | 0-100 | >80时减速 |
| 恐惧 | 0-100 | >60时逃跑优先级上升 |
| 好奇 | 0-100 | >50时会被未知吸引 |
| 攻击欲 | 0-100 | 受伤累积 |
| 思乡 | 0-100 | 远离出口太久 |

**行为 = f(AI蓝图, 需求, 环境, 记忆)**

---

# 4. 环境规则

每个环境元素有自己的规则：

| 实体 | 自己的规则 |
|------|-----------|
| 酸池 | 每秒扩散0.5m，遇墙停止 |
| 毒气 | 风向驱动，每5秒变向 |
| 火源 | 点燃可燃物，蔓延 |
| 水流 | 推动实体移动 |
| 可破坏墙 | 被撞3次倒塌 |
| 食物源 | 被吃后60秒再生 |

---

# 5. 跨Run持久化

地图是"长期场所"。Run之间发生的事会留下：

| 事件 | 持久化效果 | 消退 |
|------|-----------|------|
| 生物死亡 | 尸体保留3-5个Run | 之后被清道夫吃完 |
| 敌人死亡 | 尸体保留2-3个Run | 吸引清道夫 |
| 资源未拿走 | 保留 | 几个Run后被吃掉 |
| 走过路径 | 气味痕迹保留1-2个Run | 被嗅觉猎手追踪 |

# 6. Eco Director

根据世界状态生成事件：

```
玩家连续成功 → 增加未知威胁
玩家损失过大 → 给安全回收机会
某物种过强 → 出现天敌/疾病
污染过高 → 出现突变事件
```

事件例子：酸雨、迁徙潮、捕食者出巢、机械巡逻队经过、地下巢穴扩张、未知生物入侵

**世界不是静态地图，而是在回应你的设计。**

---

# 7. 活物密度规范

每个房间至少3-4个独立AI主体：

| 房间类型 | 最小活物数 |
|---------|-----------|
| 探索室 | 3-4 |
| 遭遇室 | 4-5 |
| 资源室 | 3-4 |
| 转场室 | 2-3 |

**低于这个密度 = 涌现不够，房间太空。**

---

# 8. M1验收标准

**同一房间播放3次，事件序列必须显著不同。**

可测量指标：
- 3次播放中至少出现5种不同"主要事件"
- 没有任何2次有80%相同的事件序列

主观验收：
- 3个非开发者各看一次，每人说出的故事都不一样
- 玩家看3次后能说"我下次想试试X，看会不会发生Y"
