# 1. 设计原则

## 1.1 房间是活物容器，不是节拍序列

每个房间是 **20-40 秒可走完**的空间。一个 Run = 3-8 个房间拼接。

设计师放活物和环境元素，不管它们什么时候互动。

| 单位 | 时长 | 活物数 |
|------|------|--------|
| 单房间 | 20-40 秒 | 3-5 个独立 AI 主体 |
| 单区域（3 房间） | 60-120 秒 | 9-15 个 |
| 完整 Run（5-7 房间） | 5-12 分钟 | 15-35 个 |

## 1.2 活物密度硬指标

按 [World_Simulation §7](World_Simulation.md#7-活物密度规范) 的密度规范，**每个房间至少包含**：

- **3-4 个独立 AI 主体**（敌人/野生生物/清道夫）
- **2-3 个环境元素**（酸池/毒气/火源/水流/可破坏墙）
- **若干静态资源**（食物/零件/宝箱）

**活物不够 = 房间太空，涌现不出来。**

## 1.3 房间不能"空"

任何房间不能有**连续 8 秒**没有任何活物在动。活物的行为自然产生事件。

# 2. 活物和环境元素

每个房间从以下元素库中配置活物和环境元素：

## 2.1 活物（独立 AI 主体）

| 类型 | 行为规则 | 涌现作用 |
|------|---------|---------|
| **巡逻者** | 守卫领地，驱逐入侵者 | 玩家是入侵者之一，不是唯一目标 |
| **捕食者** | 饿了找猎物 | 可能追玩家，可能追小生物 |
| **清道夫** | 跟着尸体/食物走 | 你杀的敌人引来清道夫 |
| **小爬虫** | 觅食，受惊逃跑 | 被吓到可能暴露藏起来的敌人 |
| **群体** | 互相靠近，领地防御 | 一个被惊动 → 其他也警戒 |
| **嗅觉猎手** | 靠气味追踪 | 生物装嗅觉腔 → 气味扩散 → 可能被误导 |

**每个房间至少 2-3 种活物。** 活物的行为规则见 [World_Simulation §2](World_Simulation.md#2-多智能体仿真)。

## 2.2 环境元素（独立规则）

| 元素 | 自己的规则 | 涌现作用 |
|------|-----------|---------|
| **酸池** | 每秒扩散 0.5m | 逼走敌人，改变地形 |
| **毒气** | 风向驱动变向 | 影响嗅觉检测范围 |
| **火源** | 点燃可燃物 | 敌人追进火源受伤 |
| **水流** | 推动实体移动 | 生物/敌人被推到意外位置 |
| **可破坏墙** | 被撞 3 次倒塌 | 改变地形，堵路/开路 |
| **食物源** | 被吃后 60 秒再生 | 吸引饥饿活物 |

**每个房间至少 1-2 种环境元素。** 环境规则见 [World_Simulation §4](World_Simulation.md#4-环境规则)。

## 2.3 静态资源

| 类型 | 作用 |
|------|------|
| **零件/宝箱** | 拾取行为 |
| **痕迹**（脚印/血迹） | 短期记忆触发，环境叙事 |
| **出口** | 任务完成或撤退 |

# 3. 房间模板定义结构

```csharp
public sealed class RoomTemplate
{
    public string TemplateId;              // R01-Lab / R02-FoyerSmell / ...
    public RoomCategory Category;          // Explore / Encounter / Resource / Transition / Anomaly
    public RoomDifficulty Difficulty;      // T1 / T2 / T3
    public Vec2Int Size;                   // 单位 m，典型 20×20
    public List<ConnectionPort> Ports;     // 出入口
    public List<AgentSpawn> Spawns;        // 活物配置（类型 + 数量范围 + 初始位置范围）
    public List<EnvEntity> EnvEntities;    // 环境元素（类型 + 位置）
    public List<StaticObstacle> Obstacles; // 墙、矮丛、岩石
    public List<ResourceSpot> Resources;   // 静态资源（零件/宝箱/食物）
}
```

**设计师定义"放什么活物"，不定义"它们什么时候互动"。**

## 3.1 房间类别

| 类别 | 节拍主调 | 出现频率 | 难度上限 |
|------|----------|----------|---------|
| **Explore** 探索室 | 无敌人，多分叉/痕迹/气味 | 40% | T1-T3 |
| **Encounter** 遭遇室 | 1-2 敌人，地形复杂 | 30% | T1-T3 |
| **Resource** 资源室 | 高价值资源 + 守卫/危险 | 15% | T2-T3 |
| **Transition** 转场室 | 窄通道/黑暗，紧张积累 | 10% | T1-T3 |
| **Anomaly** 异常室 | 损坏设备/奇怪痕迹 | 5% | T2-T3 |

## 3.2 难度等级

| Tier | 出现位置 | 敌人 | 危险地形 | 信息复杂度 |
|------|----------|------|----------|------------|
| **T1** | Run 第 1-2 房间 | 0-1 弱敌 | 无 | 低（2-3 模态混用） |
| **T2** | Run 第 3-5 房间 | 1-2 中敌 | 少量 | 中（多模态冲突） |
| **T3** | Run 第 5-7 房间 | 2-3 强敌或 1 精英 | 多处 | 高（伏击/诱饵/记忆触发） |

# 4. MVP 房间模板（10 个）

> 这是 MVP 必备的 10 个房间。M1 只用 R01。M5-M8 逐步上线其他。

## 4.1 R01-Lab — 初心房（M1 验收剧本专用）

**类别**：Explore + 弱遭遇  
**难度**：T1  
**尺寸**：20×20  
**对应**：[Sensory_Design §7 60 秒首房脚本](Sensory_Design.md#7-m1-验收剧本60-秒首房脚本)

```
            ┌─────────出口──────────┐
            │                       │
            │  [资源]    [威胁残影]  │  ← 顶部
            │   ●          ☠        │
            │                       │
            ├──墙──┐         ┌──墙──┤
            │      │ 分叉    │      │
            │      └─┐    ┌─┘      │
            │        │    │        │
            │        ●起点          │
            └────────────────────────┘
```

**节拍配置**（必出）：
- B-Fork × 1（起点正前方）
- B-Resource × 1（左路尽头）
- B-Smell-Src × 1（右路，威胁方向）
- B-Corner × 2（左右路转角）
- B-Enemy-Patrol × 1（出口走廊，弱敌）
- B-Exit × 1（顶部）

**程序变量**：
- 资源类型随机（Data / Biomass）
- 敌人血量随机 ±20%
- 威胁气味源位置在右路 3 个候选点抽取

**预想路径**：
- 主路径：起点→分叉→左路→资源→出口走廊→弱敌→出口（45-50 秒）
- 备路径：起点→分叉→右路→嗅觉警告→绕回左路→资源→出口（55-60 秒）

**节拍密度自检**：60 秒主路径走完 = 10 个事件（分叉/资源可见/拾取/绕路/嗅觉触发/敌人出现/战斗接触/胜负/继续/出口）✓

- B-Exit × 2（条件出口）

**MVP 阶段**：M3（嗅觉上线）启用。

**类别**：Transition  
**难度**：T1-T2

# 5. 程序拼接规则

## 5.1 Run 拼接算法（M8）

```
1. 抽取核心零件（Run 开始）
2. 按 Run 长度生成节点链：[T1, T1, T2, T2, T2, T3, T3]（典型 7 房间）
3. 每个节点按 Tier 从模板库抽取：
   - Tier 与房间难度匹配
   - 类别按 §3.1 频率分布抽取
   - 已抽过的模板进入冷却（同 Run 内不重复）
4. 连接：上一房间的"主出口"连接下一房间的入口
5. 验证：拼接后路径无死锁、节拍密度达标
```

## 5.2 模板变体

每个模板有 3-5 个变体，区分：

- 节拍元素位置抖动（±2m）
- 可选节拍元素的随机出现
- 敌人/资源种类的随机抽取
- 旋转 / 镜像（4 种方向）

**重玩性主要靠变体 + 拼接，不靠"完全程序化"**。

## 5.3 节点元数据（M8 星图）

```csharp
public sealed class RunNode
{
    public string RoomTemplateId;
    public int Tier;
    public NodeType Type;     // Normal / Elite / Shop / Event / Boss
    public string[] BranchTo; // 下一可选节点（M8 星图）
    public LootProfile Loot;
}
```

# 6. 节拍密度自检规范

**每个新房间模板上线前必须通过"路径自检"**：

## 6.1 主路径自检

1. 沿设计师预想的"主路径"以生物默认速度走一遍（模拟 5 m/s）
2. 标记每秒发生的事件
3. 累计"中/强/极强"事件数
4. **达标线**：单房间主路径 30 秒内至少 6 个事件

## 6.2 备路径自检

1. 沿替代路径（绕路 / 拾取 / 战斗）再走一遍
2. 同样标记节拍
3. **达标线**：备路径不能比主路径少 30%

## 6.3 失败处理

| 自检失败 | 应对 |
|---------|------|
| 主路径事件数 < 6 | 加节拍元素 |
| 连续 8 秒无事件 | 强制插入中等节拍 |
| 备路径远低于主路径 | 删除该备路径或加节拍 |
| 单房间 > 40 秒 | 拆成两个房间 |

# 8. 房间尺寸与镜头配合
