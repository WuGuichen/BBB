# MAP TEMPLATES — 地图与生态区规范

> 版本 2.0 | 2026-05-28
>
> 地图不是关卡，是**生态容器**。

---

# 1. 设计原则

## 1.1 地图是生态容器

每张地图应该有：
- 食物链
- 资源点
- 危险地形
- 巢穴
- 迁徙路线
- 昼夜变化
- 随机事件
- 敌对/中立物种

设计师放活物和环境元素，不管它们什么时候互动。

## 1.2 活物密度

每个房间至少3-4个独立AI主体 + 1-2种环境元素。

活物的行为自然产生事件，不靠设计师埋点。

# 2. 生态区

每个生态区是一张"考试卷"，测试不同的生物设计：

| 生态区 | 测试什么 | 特点 |
|--------|---------|------|
| **废弃温室** | 基础生存、路径选择 | 入门区，草食者多，少量捕食者 |
| **酸性洞穴** | 抗性、路径选择、食物策略 | 酸池扩散、清道夫多、视野差 |
| **机械废墟** | 潜行、金属采集、电子干扰 | 机械巡逻者、可拆解部件 |
| **火山地带** | 热抗性、逃跑逻辑、耐力 | 火源蔓延、高温伤害、稀有矿物 |
| **寒冷荒原** | 能量消耗、脂肪储备、迁徙行为 | 食物稀少、移动慢、群体生存 |

## 2.1 废弃温室

```
生态位：
- 草食小虫（多，弱）
- 中型捕食者（守领地）
- 清道夫（吃尸体）
- 植物（食物源）

环境：
- 普通地面
- 窄通道
- 巢穴（可筑巢）
- 水源
```

## 2.2 酸性洞穴

```
生态位：
- 酸蚀虫（抗酸，群居）
- 大型捕食者（守在酸池旁）
- 清道夫（多，吃酸蚀虫尸体）
- 寄生种（附着在大型生物上）

环境：
- 酸池（扩散）
- 黑暗区（视觉缩小）
- 窄通道
- 矿物资源点
```

## 2.3 机械废墟

```
生态位：
- 机械巡逻者（按路线，可预测）
- 小型无人机（侦察，报警）
- 异常体（规则特殊）
- 清道夫（吃机械残骸）

环境：
- 可开门/关门
- 电子干扰区
- 金属资源点
- 陷阱
```

# 3. 房间模板

```csharp
public sealed class RoomTemplate
{
    public string TemplateId;
    public RoomCategory Category;      // Explore / Encounter / Resource / Transition
    public Vec2Int Size;               // 典型20×20
    public List<ConnectionPort> Ports;
    public List<AgentSpawn> Spawns;    // 活物配置
    public List<EnvEntity> EnvEntities; // 环境元素
    public List<StaticObstacle> Obstacles;
    public List<ResourceSpot> Resources;
}
```

# 4. 活物和环境元素

## 4.1 活物

| 类型 | 行为规则 |
|------|---------|
| 巡逻者 | 守卫领地，驱逐入侵者 |
| 捕食者 | 饿了找猎物 |
| 清道夫 | 跟着尸体/食物走 |
| 小爬虫 | 觅食，受惊逃跑 |
| 群体 | 互相靠近，领地防御 |
| 机械巡逻者 | 按固定路线走 |

## 4.2 环境元素

| 元素 | 自己的规则 |
|------|-----------|
| 酸池 | 每秒扩散0.5m |
| 毒气 | 风向驱动变向 |
| 火源 | 点燃可燃物 |
| 水流 | 推动实体移动 |
| 可破坏墙 | 被撞3次倒塌 |
| 食物源 | 被吃后60秒再生 |

# 5. 程序拼接

MVP阶段：手工设计8-12个房间模板，每个有2-4个连接口。

程序生成Run时拼接：选择起点→程序选择5个房间拼接→放置出口。

在房间内程序化放置活物和环境元素，地形骨架不变。

**重玩性靠变体+拼接，不靠"完全程序化"。**
