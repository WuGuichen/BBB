# MAP TEMPLATES — 地图与生态区规范

> 版本 3.1 | 2026-05-28
>
> 地图不是关卡，是**生态容器**。每个生态区是Run中的一个节点。
> 活物配置只携带"名词定义"（EmitterDef + SensorDef + 性格权重），不携带事件/节拍。
>
> 冲突裁决：System_Core > 本文档。

---

# 1. 设计原则

## 1.1 Run结构中的地图

```
Run = 3-5个连续生态区
每个生态区 = 3-5个房间（随机拼接）
每个房间 = 地形 + 活物 + 环境元素
```

**每次Run的房间组合不同。** 程序化拼接保证重玩性。

## 1.2 活物密度

每个房间至少3-4个独立AI主体 + 1-2种环境元素。

---

# 2. 生态区

每个生态区是一张"考试卷"，测试不同的生物设计：

| 生态区 | 测试什么 | Run位置 | 难度 |
|--------|---------|---------|------|
| **废弃温室** | 基础生存、路径选择 | 第1区 | T1 |
| **酸性洞穴** | 抗性、路径选择 | 第2区 | T2 |
| **机械废墟** | 潜行、电子干扰 | 第3区 | T2-T3 |
| **火山地带** | 热抗性、逃跑逻辑 | 第4区 | T3 |
| **异常区** | 未知规则 | 第5区 | T3+ |

## 2.1 废弃温室

```
生态位：
- 草食小虫（多，弱）
- 中型捕食者（守领地）
- 清道夫（吃尸体）

环境：
- 普通地面
- 窄通道
- 巢穴
- 水源
```

## 2.2 酸性洞穴

```
生态位：
- 酸蚀虫（抗酸，群居）
- 大型捕食者（守在酸池旁）
- 清道夫

环境：
- 酸池（扩散）
- 黑暗区（视觉缩小）
- 矿物资源点
```

## 2.3 机械废墟

```
生态位：
- 机械巡逻者（按路线，可预测）
- 小型无人机（侦察，报警）
- 异常体（规则特殊）

环境：
- 可开门/关门
- 电子干扰区
- 金属资源点
```

# 3. 房间模板

```csharp
public sealed class RoomTemplate
{
    public string TemplateId;
    public RoomCategory Category;      // Explore / Encounter / Resource / Transition
    public Vec2Int Size;               // 典型20×20
    public List<ConnectionPort> Ports;
    public List<AgentSpawn> Spawns;    // 活物配置
    public List<EnvEntity> EnvEntities; // 环境元素
    public List<StaticObstacle> Obstacles;
    public List<ResourceSpot> Resources;
}
```

---

# 4. 程序拼接

每个生态区由3-5个房间模板拼接：

```
1. 选择入口房间
2. 程序选择2-3个中间房间（随机）
3. 选择出口房间
4. 连接：上一房间出口连接下一房间入口
5. 在房间内程序化放置活物和环境元素
```

**重玩性靠变体+拼接。** 地形固定，活物和环境随机。

---

# 5. 撤退点

每个生态区出口前有撤退选项：
- 选择"继续" → 进入下一个生态区
- 选择"撤退" → Run结束，带回所有收集的资源

**撤退是策略核心。** 继续深入风险更高但奖励更好。
