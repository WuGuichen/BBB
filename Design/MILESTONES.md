# MILESTONES — 制作计划

> 版本 0.9 | 2026-05-27

---

# 1. 阶段总览

**M1-M2 是判生死节点。** 如果一只生物在迷雾房间里走 60 秒，找东西、犹豫、被你喊一句话改变了方向 — 如果这一分钟让 playtest 玩家睁大眼睛，项目就成立。

| 阶段 | 目标 | 核心验证 |
|------|------|----------|
| **M0** | 骨架 + Sensory Sandbox | RuntimeHost + 占位生物 + 可重置调试场景 |
| **M1** ⚠️ | 感知与迷雾 | 一只生物的传感器揭开未知（验收剧本：Sensory §7） |
| **M2** ⚠️ | 性格 + 犹豫 + AI 四层反馈 | 矛盾叠加产生犹豫，性格可读，玩家愿意再派它 |
| **M3** | 嗅觉模态 + Trust v0 表现层 | 嗅觉视觉差异 + "它点了点头" |
| **M4** | 干预动词 + 短期记忆 | 4 个喊话动词性格敏感（固定供给，不测稀缺） |
| **M5** | 意外与遭遇 + 伤痕/死亡纪念 | 战斗是事件不是高潮，生物有"角色化"留痕 |
| **M6** | Trust v1 系统层 | 信任度真正影响 AI 决策 |
| **M7** | 零件 + Run 结构 + Mission Report | 探索获得零件，回放叙事 |
| **M8** | 多区域 Run + Run 总结 | 完整 Roguelite 闭环 |

> **v0.9 变更**：多生物从 milestone 移除（详 GDD §5），改为 Post-MVP 独立决策。Trust 拆为 v0 表现层（M3）+ v1 系统层（M6）。零件经济推到 M7（之前是 M2-M4 混在一起）。

# 2. 各阶段详情

## M0：骨架 + Sensory Sandbox

要做：
1. `Assets/Scripts/WGame/` + `WGameRuntimeCompositionRoot`（游戏层组合根，基于 MxFramework）
2. `RuntimeHost` + `RuntimeCommandBuffer`（复用框架）
3. AABB 房间 + 占位生物 + 基础移动
4. **Sensory Sandbox 调试场景**：固定房间、可重置、无限信号、可热切 build。详见 [TECHNICAL §10](TECHNICAL.md)
5. 2.5D 等距相机基线（55-60°，正交或弱透视）
6. 滞后跟随相机（200-300ms 延迟）

验收：
- 生物可移动，不穿墙，Replay hash 一致
- Sandbox 场景一键重置
- 相机角度固定，跟随生物，滞后 200-300ms

## M1：感知与迷雾 ⚠️ 判生死

**目标**：一只生物的传感器揭开未知，玩家通过它的眼睛看世界。

**验收剧本**：[Sensory_Design §7 — 60 秒首房脚本](Sensory_Design.md#7-m1-验收剧本60-秒首房脚本)。**M1 完成的定义就是能跑通这个 60 秒**。

**房间**：[Map_Templates R01-Lab](Map_Templates.md#41-r01-lab--初心房m1-验收剧本专用)（M1 唯一房间）。

要做：
1. 战争迷雾（Unity 视觉层，生物走到哪里揭开哪里）
2. 视觉传感器（CombatPhysicsWorld Sphere 查询 → 揭开圆形范围）
3. 已揭开区域的"记忆衰减"视觉（[Sensory §1.1](Sensory_Design.md#11-已揭开区域的记忆衰减)）
4. R01-Lab 房间布局（按 Map_Templates §4.1 节拍配置）
5. 行为层基础状态（探索/警戒/拾取/前进/后退），通过物理动作即可读
6. 信息密度保证：每 5-10 秒至少一个新信息（按 [Map_Templates §6](Map_Templates.md#6-节拍密度自检规范) 自检）

验收：
- 跑通 Sensory §7 剧本前 30 秒（嗅觉触发后的内容延到 M3）
- 信息密度：60 秒内至少 7 个信息事件
- **主观 Playtest**：3 个非开发者看 60 秒后说"我想知道前面有什么"

**诊断问题**（必问）：
- "你觉得它刚才在做什么？"
- "你想看到它再走一次吗？"

**不通过则停**。这不是"加功能"问题，是核心动词是否成立的问题。

## M2：性格 + 犹豫 + AI 四层反馈 ⚠️ 判生死

**目标**：AI 模块改变行为，矛盾叠加产生犹豫，性格一句话可读，玩家愿意再派同一只生物出去。

要做：
1. AI 模块（勇气核心 / 谨慎核心 / 贪欲核心）+ 矛盾叠加
2. **性格三表现规格**（[GDD §4.3](GDD.md#43-性格的三类表现规格-mvp)）：每个模块定义微行为 + UI 语言 + 音效倾向
3. **AI 反馈四层**（[Sensory §4](Sensory_Design.md#4-ai-反馈四层)）：
   - 行为层：状态切换的物理表现
   - 情绪层：头顶图标 `?` `!` `★`
   - 理由层：底部行为日志（含未发生选择）
   - 结果层：占位文本（信号反馈，但 Trust 变量延到 M3）
4. 犹豫表现（[Sensory §5](Sensory_Design.md#5-犹豫的表现规范)）
5. **生物角色化最小项**（[GDD §13](GDD.md#13-生物作为角色-mvp)）：命名 + 归来反应

验收：
- 关掉 UI 和音效，只看物理动作，3 个非开发者能说出"它是哪种性格"
- 勇气+谨慎装一起，可见血量分段行为（高冲/中犹豫/低逃）
- 60 秒内至少 2 个性格高亮事件 + 1 个犹豫时刻
- 犹豫时刻 Brain View 浮现两个候选 + 主导零件

**诊断问题**（必问）：
- "你觉得它刚才为什么停下？"
- "你觉得它害怕了吗？为什么？"
- "如果它死了，是你判断错了、它性格导致的，还是系统随机？"
- **"你愿不愿意再派同一只生物出去？"** ← 关键

**不通过则停**。如果玩家说"换一只也行"，[GDD §13](GDD.md#13-生物作为角色-mvp) 需要加强。如果玩家说"它就是随机走"，性格表现层不合格。

## M3：嗅觉模态 + Trust v0 表现层

**目标**：第二种感知有清晰的视觉差异；玩家能感受到"它听了 / 没听 / 迟疑"。

要做：
1. 嗅觉传感器（检测范围内标记实体 → 生成气味流）
2. 气味流视觉语言（红/绿色气流，±30° 误差，3 档距离）
3. 嗅觉 AI fact（`threat.smell` confidence 0.5-0.7）
4. **跑通 Sensory §7 剧本的完整 60 秒**（含嗅觉触发）
5. **新增房间** [R02-FoyerSmell](Map_Templates.md#42-r02-foyersmell--嗅觉门厅)（嗅觉验证专用）
6. **Trust v0 表现层**（[GDD §4.5](GDD.md#45-信任度) + [Sensory §4.4](Sensory_Design.md#44-结果层-mvp--看懂关系记忆)）：
   - 内部 0.0-1.0 变量（不影响 AI 决策）
   - 信号反馈四种短语（听了/受伤/没听/坑了）
   - 犹豫图标变体（`?` vs `?!`）

验收：
- 嗅觉和视觉的显示方式明显不同
- 嗅觉早于视觉触发的节拍可见（剧本 30-40 秒段）
- 每次玩家信号都有结果层反馈，没有"按了没反应"
- **主观 Playtest**：3 个非开发者
  - 能区分"看到敌人"和"闻到敌人"
  - 描述信号后生物的反应（"它点了点头" / "它没听" / "它好像不太高兴"）

**诊断问题**：
- "你觉得它听见你了吗？"
- "你觉得它是没听见，还是听见了但不想听？"
- "你信任它现在听你的吗？"

## M4：干预动词 + 短期记忆

**目标**：4 个喊话动词，性格敏感；生物记得最近的遭遇。

要做：
1. 警告 / 建议 / 安抚 / 鼓舞 四个动词
2. 性格抵抗系数（勇气抵抗安抚，谨慎抵抗鼓舞）
3. **固定供给：每次投放 2-3 次喊话，无道具栏**（[GDD §6.2](GDD.md#62-干预资源两段经济-mvp)：不测稀缺性）
4. 短期记忆模块（被攻击过的方向、发现过资源的位置）
5. 记忆触发：进入"危险方向"时朝向锁定（[GDD §13 记忆方向](GDD.md#13-生物作为角色-mvp)）
6. **新增房间** [R03-Corridor](Map_Templates.md#43-r03-corridor--转场走廊)（记忆触发演示）

验收：
- 警告让谨慎核心绕路，但勇气核心可能忽略（明显的抵抗）
- 安抚让犹豫的生物冷静下来，但纯勇气型不受影响
- 重复投放同一只生物到同一房间，第二次比第一次更警惕特定方向
- **主观 Playtest**：3 个非开发者觉得"我在和它对话"，且能描述至少 2 种动词的不同响应

**诊断问题**：
- "你的警告有没有用？"
- "它好像记住了什么吗？"

---

## M5：意外与遭遇 + 角色化留痕

**目标**：战斗是探险的一个事件，生物有"角色化"留痕。

要做：
1. 敌人（巡逻 → 发现生物 → 追击）
2. 接触伤害（3-5 秒战斗，0.3 秒前摇 + 音效 + 伤害数字）
3. 战斗中的 AI 四层反馈（[GDD §9.2](GDD.md#92-战斗中的反馈)）
4. **伤痕表现**：受过重伤后占位体留色块/缺口，跨投放保留（[GDD §13](GDD.md#13-生物作为角色-mvp)）
5. **死亡纪念**：屏幕中央 2 秒"{名字} 在第 N 个房间倒下了"
6. 死亡时记忆方向 +1（同伴死亡的位置）
7. **新增房间** [R06-Ambush](Map_Templates.md#46-r06-ambush--伏击点)（伏击演示战斗事件性）

验收：
- 战斗在 5 秒内结束，结果可读
- 勇气核心主动打，谨慎核心绕开，叠加时血量分段
- 战斗不是高潮 — **主观 Playtest**："战斗是探险的一部分，不是重点"
- 死亡有情感重量 — **主观 Playtest**："我有点难过它死了"

**诊断问题**：
- "战斗是这游戏的重点吗？"
- "它死了你觉得怎么样？"

---

## M6：Trust v1 系统层

**目标**：信任度真正影响 AI 决策（不只是表现）。

要做：
1. Trust 进入 AiWorldState fact（`player.trust`）
2. 信号效果系数随 Trust 变化（高信任 → shift 倍率 ×1.5，低信任 → ×0.5，极低 → 完全忽略）
3. 信任度获得/损失规则（[GDD §4.5](GDD.md#trust-v1系统层-post-mvp-m6)）
4. Run 总结里展示 Trust 变化曲线

验收：
- 救过它之后，下次犹豫时更倾向听你的（Goal priority shift 倍率提升可观测）
- 坑过它若干次后，信号几乎无效
- 同一房间用同一生物：信号成功的 Run 比信号失败的 Run，第二次任务行为差异明显
- **主观 Playtest**：3 个非开发者能说出"它现在不太听我的了"

**诊断问题**：
- "它最近变了吗？"
- "你觉得你们俩关系怎么样？"

---

## M7：零件 + Run 结构 + Mission Report

**目标**：零件经济上线，Mission Report 提供认知反馈。

要做：
1. 零件掉落（宝箱/敌人掉落，随机性）
2. 背包（8 格，满时丢弃 UX）
3. 投放前组装界面（选身体部件 + AI 模块）
4. 核心零件（Run 开始选，死亡不丢）
5. **信号道具化**（[GDD §6.2](GDD.md#62-干预资源两段经济-mvp)）：M4 的固定喊话改为道具栏，一次性，0-3 个/局
6. **Mission Report**（[GDD §10.4](GDD.md#104-mission-report-mvp-m6)）：生物视角的回放叙事
7. 连续房间 Run 结构（3 个房间，按 [Map_Templates §5.1 拼接算法](Map_Templates.md#51-run-拼接算法m8)）
8. **新增房间** [R04-Cache](Map_Templates.md#44-r04-cache--资源密室)（资源经济演示）

验收：
- 每局零件不同 → 生物性格不同 → 体验不同
- Mission Report 能解释这局发生了什么，玩家能说出"下次该改什么"
- 信号稀缺时玩家会犹豫"什么时候用" → 出现策略瞬间

**诊断问题**：
- "下一局你想试试什么？"（如果能说出具体的，认知循环成立）
- "你愿意继续玩下一个 Run 吗？"

---

## M8：多区域 Run + Run 总结

**目标**：完整 Roguelite 闭环。

要做：
1. 5-7 节点的星图结构（精英/事件/商店/出口）
2. 跨区域零件保留 / 部分丢失规则
3. Run 总结（[GDD §13](GDD.md#13-生物作为角色-mvp)）：一句性格总结 + 关键决策时间线
4. 死亡时生物的最终命运记录（"它死在第 X 个房间，记得 Y 个危险方向"）
5. **新增房间** [R05-Junction](Map_Templates.md#45-r05-junction--三岔路口) + [R10-Exit](Map_Templates.md#410-r10-exit--终点房)
6. 程序拼接（按 [Map_Templates §5](Map_Templates.md#5-程序拼接规则)）+ 模板变体抽取

验收：
- 完整 Run 15-25 分钟
- 一句性格总结准确：3 个非开发者能从总结中认出自己的生物
- **主观 Playtest**：完成一次 Run 后想再来一次

**诊断问题**：
- "用一句话描述这只生物"
- "下次你想培养一只什么样的"

# 3. 最小竖切

| 类别 | 内容 |
|------|------|
| **生物** | **1 只**（v0.9 锁定），HP/MoveSpeed/Attack，可命名 |
| **感知** | 视觉（圆形揭开）+ 嗅觉（气流方向） |
| **AI 模块** | 勇气 / 谨慎 / 贪欲（可叠加） |
| **干预** | 警告 / 建议 / 安抚 / 鼓舞（M4 固定供给，M7 道具化） |
| **零件** | 快腿/厚甲/毒牙/嗅觉腔/三种 AI 核心 |
| **地图** | 10 个手工房间模板（见 [Map_Templates.md](Map_Templates.md)），程序拼接，单房间 20×20m |
| **UI** | 行为日志 + 犹豫提示 + 信号按钮 + 头顶状态 |
| **角色化** | 命名 + 伤痕 + 归来反应 + 死亡纪念 + Run 总结 |
| **反馈** | AI 四层反馈 + Mission Report 回放叙事 |
| **调试** | Sensory Sandbox + Decision Trace Overlay |

# 4. Playtest 节点

每个阶段除主观打分外，必须问**诊断题**。"好不好玩"不算诊断。

| 阶段 | 主要 Playtest 问题 | 诊断题 | 不通过则 |
|------|----------|--------|----------|
| **M1** | "你想知道前面有什么吗？" | 你觉得它刚才在做什么？想再看一次吗？ | 停下，重做行为层 |
| **M2** | "你愿意再派同一只出去吗？" | 它为什么停下？害怕了吗？是性格还是随机？ | 停下，加强性格表现 / 角色化 |
| **M3** | 能区分"看到"和"闻到"吗？ | 它听见你了吗？信任你吗？ | 感知/Trust v0 要改 |
| **M4** | "我在和它对话吗？" | 它记住了什么吗？警告有用吗？ | 干预动词要改 |
| **M5** | 战斗是重点吗？ | 它死了你怎么想？ | 战斗太抢眼 / 角色化不足 |
| **M6** | "它不太听我的了"？ | 它最近变了吗？ | Trust 系统层效果要更明显 |
| **M7** | 下一局想试什么？ | 报告解释了这局吗？ | Mission Report 要改 |
| **M8** | 完成 Run 想再来？ | 用一句话描述这只生物 | Run 结构 / 总结要改 |

**关键诊断题（贯穿全程）**：

> **"你愿不愿意再派同一只生物出去？"**

如果玩家想再派同一只 = 陪伴感成立。
如果想换一只 = 角色化不够。
如果不愿意 = AI 可读性不够。

# 5. 风险

| 风险 | 应对 |
|------|------|
| 探索节奏太慢 | 地图小而密，信息事件每 5-10 秒（Sensory §6） |
| 看不懂生物在想什么 | AI 四层反馈 + M1/M2 严格 Playtest |
| 战斗抢走注意力 | 短而突然（3-5 秒），战斗不是 milestone 主题 |
| 陪伴感立不起来 | Trust v0 表现层提到 M3 + 角色化提到 M2 |
| 性格可预测就无聊 | 短期记忆 + 矛盾叠加 + 环境敏感 |
| 感知可视化成本高 | MVP 只做视觉+嗅觉，听觉 Post-MVP |
| 稀缺性和沟通感混在一起验证 | M4 不测稀缺，M7 才道具化 |

---

# 6. 暂缓

| 功能 | 原因 |
|------|------|
| 多生物小队 | v0.9 锁定：MVP 单生物，多生物是 Post-MVP 独立决策 |
| 听觉模态 | 先验证视觉+嗅觉 |
| Trust v1 系统层 | M6 才做（v0 表现层在 M3） |
| 询问 / 指认动词 | Post-MVP |
| 分支星图 | M8 才做（M7 先做连续房间） |
| 零件升级 | Post-MVP |
| 生态 / Mod / 多人 | 不在 MVP |
