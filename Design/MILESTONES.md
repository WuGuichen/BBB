# MILESTONES — 制作计划

> 版本 0.7 | 2026-05-27

---

# 1. 阶段总览

| 阶段 | 目标 | 核心验证 |
|------|------|----------|
| **M0** | 骨架 | RuntimeHost + 占位生物 + AABB 房间 |

| **M2** | 零件掉落 + 背包 | 探索发现零件，拾取入背包 |

# 2. 各阶段详情

## M0：骨架

要做：
1. `Assets/Scripts/VVV/` + `VVVRuntimeCompositionRoot`
2. `RuntimeHost` + `RuntimeCommandBuffer`
3. AABB 房间 + 占位生物

验收：Play 后有生物，Frame 增长，Reset 后状态一致。

---

## M1：单房间自动行动

复用：`CombatPhysicsWorld` + `CombatKinematicMotor`

要做：
1. 地图（AABB 墙壁 + 资源点 + 出口）
2. 生物自主移动到目标
3. 碰撞不穿透

验收：能到达目标，不穿墙，Replay hash 一致。

---

## M2：零件掉落 + 背包

要做：
1. `PartDefinition`（白色：快腿/厚甲/毒牙/勇气核心/谨慎核心/贪欲核心）
2. 零件散落地图（宝箱位置）
3. 拾取入背包
4. `PlayerInventory`（8 格，满时提示丢弃）
5. 背包 UI

验收：能发现并拾取零件。背包满时不能继续拾取。

**M2 完成后立即给人看**（哪怕只是 30 秒视频）：零件拾取是否有趣？背包管理是否有取舍？

---

## M3：组装 + AI 模块 + 小队

要做：
1. 投放前组装界面（选 2 只生物的部件 + AI 模块）
2. `AiModuleDefinition`（勇气/谨慎/贪欲，效果大幅）
3. **2 只小队槽位**，同时投放
4. 队形（前排/后排）
5. AI 模块矛盾叠加（勇气+谨慎 = 高血量冲低血量逃）

验收：
- 装勇气核心：看到敌人主动进攻
- 装谨慎核心：看到敌人绕路
- 装贪欲核心：偏离目标去捡资源
- **勇气+谨慎叠加时出现犹豫**（中间血量摇摆）
- **小队冲突可见**（一只冲一只跑）
- **主观 Playtest**：3 个非开发者能说出"这两只性格不一样"

---

## M4：敌人 + 自动战斗

要做：
1. 敌人（巡逻 + 追击）
2. 接触伤害 + **0.3 秒前摇 + 音效 + 伤害数字弹出**
3. HP 系统
4. 死亡 → 尸体
5. 敌人掉落零件

验收：
- 战斗可在 1 秒内看懂（谁打了谁、扣了多少血）
- 勇气核心生物主动打，谨慎核心绕开
- 勇气+谨慎叠加时：高血量冲、低血量逃
- 死亡时失去非核心零件，核心零件保留
- **性格在战斗中最明显**（"它残血了还在打！"）
- **主观 Playtest**：3 个非开发者能说出"这场战斗谁赢了为什么"

---

## M5：信号 + 侦察 + 撤退

要做：
1. 信号道具（召回信标 / 危险标记 / 鼓励脉冲）— 探索中获得
2. 侦察探针 — 使用时揭开局部地图
3. 撤退指令 — 不需道具但节点视为失败
4. 投放时选择携带哪些（信号槽 2 个）
5. 犹豫提示（"？" + 画面轻微放慢）

验收：
- 信号/侦察/撤退构成 3 个操作维度
- 每次投放 20-40 秒内至少 2-3 次有意义操作
- **决策密度 ≥ 4 次/分钟**
- 信号是稀缺的（可能一局找到 0-3 个）
- **主观 Playtest**：3 个非开发者觉得"任务中我一直在做决定"

---

## M6：单层星图

要做：
1. 星图 5-7 节点，玩家选路径
2. 节点类型：普通 / 精英（强敌人好零件）/ 事件（随机效果）/ 休息（恢复）
3. 零件跨节点保留
4. 核心零件死亡不丢

验收：
- 路径选择有意义（精英高风险高收益）
- 死亡有代价但不挫败（核心零件还在）
- **主观 Playtest**：3 个非开发者选路径时有犹豫（"走精英还是走安全的？"）

---

## M7：多层 Run

要做：
1. 3 层星图，难度递增
2. Run 总结（用了哪些零件、哪些决策关键）
3. Build Identity 可感知

验收：
- 一次完整 Run 10-15 分钟
- 第 3 层时玩家能说出"我这局是 X 流"
- **主观 Playtest**：3 个非开发者完成 Run 后想再试一次，且想试不同的起始特征

---

## M8：反馈产品化

要做：
1. 性格高亮（做了什么 + **没做什么**）
2. 犹豫提示优化
3. 战斗伤害数字/暴击/未命中
4. 结果展示（性格标签 + 获得/失去零件 + 小队状态）
5. Run 总结（关键决策、Build 分析）

验收：
- 90% 失败能归因到一个主要原因
- 性格高亮覆盖"做了"和"没做"的决策
- 战斗结果在 1 秒内可读

# 5. 风险

| 风险 | 应对 |
|------|------|
| 决策密度太低 | 小队制 + 3 操作维度 + 分支星图 |
| 单体无协同 | 2 只小队，性格互补/冲突 |
| 接触伤害没可观察性 | 前摇 + 音效 + 伤害数字 |
| 没有 Build Identity | 生物持续存在，核心零件不丢 |
| 性格可预测就无聊 | AI 模块矛盾叠加产生犹豫 |
| 零件随机挫败 | 核心零件死亡不丢 |
| 确定性 hash 回归 | Replay hash CI 自动化 |

---

# 6. 暂缓

| 功能 | 原因 |
|------|------|
| 3 只小队 | 先验证 2 只 |
| 复杂攻击类型 | 先做接触伤害 |
| 程序化地图 | 先手工设计 |
| 零件升级/合成 | 先验证核心循环 |
| 永久进度 | 先做单 Run |
| 生态模拟 | 不在 MVP |
| 多人 / Mod | 不在 MVP |
| 正式美术 | 占位体 |
