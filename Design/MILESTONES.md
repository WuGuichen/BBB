# MILESTONES — 制作计划

> 版本 0.6 | 2026-05-27

---

# 1. 阶段总览

| 阶段 | 目标 | 核心验证 |
|------|------|----------|
| **M0** | 骨架 | RuntimeHost + 占位生物 + AABB 房间 |
| **M1** | 单房间自动行动 | 生物自主移动到目标点，Replay hash 一致 |
| **M2** | 零件掉落 + 背包 | 探索中发现零件，拾取入背包 |
| **M3** | 组装 + AI 模块 | 用手里的零件组装生物，AI 模块改变行为 |
| **M4** | 敌人 + 自动战斗 | 接触伤害，HP，死亡，性格在战斗中体现 |
| **M5** | 信号道具 | 探索中获得信号，关键时刻使用 |
| **M6** | 单区域 Roguelike | 完整的"探索→获得→组装→投放→结果"闭环 |
| **M7** | 多区域 Run | 3 个区域连续挑战，零件跨区域保留/丢失 |
| **M8** | 反馈产品化 | 性格高亮、犹豫提示、结果展示、Run 总结 |

---

# 2. 各阶段详情

## M0：骨架

要做：
1. `Assets/Scripts/VVV/` + `VVVRuntimeCompositionRoot`
2. `RuntimeHost` + `RuntimeCommandBuffer`
3. AABB 房间 + 占位生物

验收：Play 后有生物，Frame 增长，Reset 后状态一致。

---

## M1：单房间自动行动

复用：`CombatPhysicsWorld` + `CombatKinematicMotor`

要做：
1. 地图定义（AABB 墙壁 + 资源点 + 出口）
2. 生物自主移动到目标点
3. 碰撞不穿透
4. 到达资源点 → 拾取 → 移动到出口 → 完成

验收：不穿墙，能到达目标，Replay hash 一致。

---

## M2：零件掉落 + 背包

要做：
1. `PartDefinition`（白色快腿/厚甲/毒牙/勇气核心/谨慎核心）
2. 零件散落在地图上（宝箱位置）
3. 生物经过宝箱 → 拾取零件 → 入背包
4. `PlayerInventory`（8 格背包，满时提示丢弃）
5. 背包 UI

验收：探索中能发现并拾取零件。背包满时不能继续拾取。

---

## M3：组装 + AI 模块

要做：
1. 投放前组装界面（选身体部件 + AI 模块）
2. `AiModuleDefinition`（勇气核心 / 谨慎核心 / 贪欲核心）
3. AI 模块→Goal priority / Action cost 映射
4. 不同模块组装后行为明显不同

验收：
- 装勇气核心：生物看到敌人主动进攻
- 装谨慎核心：生物看到敌人绕路
- 装贪欲核心：生物偏离目标去捡额外资源
- **性格一句话可描述**（"这是一只好斗的猎手"）

---

## M4：敌人 + 自动战斗

要做：
1. 敌人（巡逻 + 发现玩家后追击）
2. 接触伤害（overlap → 扣 HP）
3. HP 系统（生物和敌人都有 HP）
4. 死亡 → 尸体
5. 敌人掉落零件

验收：
- 勇气核心生物会主动打敌人
- 谨慎核心生物会绕开敌人
- 贪欲核心生物可能边打边捡
- 死亡时失去本次获得的零件
- **性格在战斗中最明显**（"它残血了还在打！"）

---

## M5：信号道具

要做：
1. 信号道具（召回信标 / 危险标记 / 鼓励脉冲）
2. 信号在探索中获得（宝箱掉落）
3. 投放时选择携带哪些信号（信号槽，最多 2 个）
4. 任务中使用信号 → 改变 AI Utility 分数
5. 信号使用 UI

验收：
- 信号是稀缺的（可能一局找到 0-3 个）
- 使用召回信标能让生物中途返回
- **没有信号时纯靠设计，有信号时多一层策略**

---

## M6：单区域 Roguelike

要做：
1. 完整单区域流程：探索→获得→组装→投放→结果
2. 地图：多条路线 + 宝箱位置 + 敌人巡逻 + 出口
3. 任务目标（到达资源点 + 返回出口）
4. 死亡/成功判定
5. 结果展示（获得/失去零件 + 性格标签）

验收：
- 完整玩一局 2-5 分钟
- 每局零件不同 → 策略不同
- 成功有成就感（"我用这些零件拼出来了"）
- 失败有学习感（"下次遇到这种敌人应该用谨慎核心"）
- **主观 Playtest**：3 个非开发者玩一局后想再试一次

---

## M7：多区域 Run

要做：
1. 3 个区域连续挑战（难度递增）
2. 零件跨区域保留
3. 死亡失去本次获得的零件
4. 区域选择界面
5. Run 总结

验收：
- 3 个区域难度递进
- 死亡有代价但不挫败（"我还有之前的零件"）
- 一次完整 Run 10-15 分钟
- **主观 Playtest**：3 个非开发者完成一次 Run 后能说出"我下一局想试试不同的零件组合"

---

## M8：反馈产品化

要做：
1. 性格行为高亮（"它选择了进攻（勇气核心）"）
2. 犹豫提示（"？" + 画面轻微放慢）
3. 结果展示（性格标签 + 获得/失去零件）
4. Run 总结（哪些决策关键、哪些零件发挥了作用）

验收：
- 性格高亮在关键决策时出现
- 犹豫提示在 AI 不确定时可见
- 结果展示一句话说清成败
- **90% 失败能归因到一个主要原因**

# 3. 最小竖切清单

| 类别 | 内容 |
|------|------|
| **地图** | 1 个区域：多条路线 + 2-3 个宝箱 + 1-2 个敌人 + 资源点 + 出口 |
| **零件** | 身体：快腿/厚甲。AI：勇气/谨慎/贪欲。信号：召回信标 |
| **生物** | HP / MoveSpeed / Attack / Defense |
| **敌人** | 巡逻 + 追击 + 接触伤害 |
| **背包** | 8 格 |
| **UI** | 组装界面 / 背包 / 信号按钮 / 结果展示 |

# 4. Playtest 节点

| 阶段 | 内容 |
|------|------|
| M3 | 能感受到不同 AI 模块的行为差异 |
| M6 | 玩一局后想再试一次 |
| M7 | 完成一次 Run 后能说出策略思路 |
| M8 | 能根据结果说出"下次我该换什么零件" |

---

# 5. 风险

| 风险 | 应对 |
|------|------|
| 零件池太小重复感强 | MVP 5-8 种零件 + 稀有度变体 |
| 战斗太简单 | MVP 先验证核心循环；Post-MVP 加攻击类型 |
| 间接控制不如手操 | 手操做不了 — 零件随机，策略才是核心 |
| 单局太短没深度 | M7 多区域 Run 拉长 |
| 地图内容不够 | 每区域手工设计多条路线 + 宝箱分布 |

---

# 6. 暂缓

| 功能 | 原因 |
|------|------|
| 程序化地图生成 | 先手工设计验证核心循环 |
| 复杂攻击类型 | 先做接触伤害 |
| 生态模拟 | 先验证 Roguelike 循环 |
| 玩家 AI 编辑器 | 先用 AI 模块（勇气/谨慎/贪欲） |
| 永久进度 | 先做单 Run |
| 多人 / Mod | 不在 MVP |
| 正式美术 | 占位体 |
